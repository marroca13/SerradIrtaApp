О программе ArcGIS Web Adaptor

ArcGIS Web Adaptor (IIS) является компонентом Portal for ArcGIS, который поддерживает интеграцию вашего портала с имеющимся веб-сервером и механизмами обеспечения безопасности вашей организации. ArcGIS Web Adaptor необходим для использования аутентификации Windows на вашем портале..

Web Adaptor – это приложение, работающее на вашем веб-сайте и отправляющее запросы к компьютеру, на котором установлен Portal for ArcGIS. Установка IIS совместима с версиями IIS на Windows 7, 8, 8.1, 10, Server 2008, 2008 R2, Server 2012 и 2012 R2. Это соответствует версиям IIS 7.0, 7.5, 8.0, 8.5 и 10

Преимущества Web Adaptor

Web Adaptor предоставляет ряд преимуществ:

  • Позволяет интегрировать Portal for ArcGIS с существующим веб-сервером организации. Включив веб-сервер в свой сайт, вы получаете возможность разместить веб-приложения, использующие сервисы ГИС.

  • Предоставляет вам возможность использовать хранилище учетных записей вашей организации, а также политику безопасности на сетевом уровне. Например, при работе с IIS вы можете использовать Встроенную аутентификацию Windows для ограничения круга лиц, входящих на портал.Также вы можете воспользоваться Инфраструктурой открытых ключей (PKI) либо другим хранилищем учетных записей, для которого веб-сервер имеет встроенную или расширяемую поддержку. Кроме того, вы сможете обеспечить единую регистрацию или другую пользовательскую аутентификацию при входе – для работы с сервисами, веб-приложениями и Portal for ArcGIS.

  • Позволяет вам открыть Portal for ArcGIS, используя другое имя сайта, а не используемое по умолчанию имя arcgis.

  • Позволяет открыть Portal for ArcGIS через порт 80 или 443.

    Примечание:

    Вы можете использовать для Web Adaptor только порт 80 или 443. Другие порты не поддерживаются.

Сценарий развертывания Web Adaptor

Версия Web Adaptor должна всегда совпадать с версией Portal for ArcGIS. Если вы настроили множество Web Adaptor для одного веб-сервера, то вы можете использовать различные номера версий для всех веб-адаптеров.

Прежние версии:

Для версий 10.2 и более ранних от всех Web Adaptor на одном веб-сервере требовалось использование одинакового номера версии; запуск установки обновлял все существующие Web Adaptor до последней версии.

Web Adaptor не зависит от платформы Portal for ArcGIS, поэтому развертываемый Web Adaptor не обязательно должен соответствовать платформе операционной системы вашего портала. Например, если ваш портал работает на Linux, вы можете развернуть ArcGIS Web Adaptor (IIS) для работы с Portal for ArcGIS.

Осуществление доступа к порталу с установленным Web Adaptor

После установки и настройки Web Adaptor, используемый для доступа к вашему порталу будет иметь формат https://webadaptorhost.domain.com/webadaptorname/home. Например, если компьютер с Web Adaptor называется wa, находится в домене myorg.net, а сам Web Adaptor называется arcgis, доступ к порталу осуществляется по URL-адресу https://wa.myorg.net/arcgis/home.

Если вы настроили портал на использование HTTPS для передачи данных, обновите URL ярлыков веб-сайта портала и справки с учетом использования протокола https вместо http, иначе вы увидите сообщения об ошибке в браузере при попытке перейти по старым URL-ссылкам ярлыков.

Интерфейс настройки Web Adaptor

В Web Adaptor имеется собственное руководство по настройке и установке, которое устанавливается отдельно от Portal for ArcGIS. Web Adaptor необходимо установить на компьютере, на котором запущен веб-сервер. Это может быть компьютер, на котором уже работает Portal for ArcGIS или отдельный компьютер.

Более подробно об установке см. в разделе Обзор использования Web Adaptor с Portal for ArcGIS.

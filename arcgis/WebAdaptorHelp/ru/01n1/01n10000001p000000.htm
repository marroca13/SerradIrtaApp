Настройка ArcGIS Web Adaptor

После установки ArcGIS Web Adaptor, его необходимо настроить для работы с ArcGIS Enterprise. Это делается с помощью веб-страницы конфигурации или утилиты командной строки, которая устанавливается вместе с ArcGIS Web Adaptor. В целях безопасности доступ к странице конфигурации имеется только на компьютере, где установлен ArcGIS Web Adaptor.

После открытия страницы конфигурации отображается текущий статус ArcGIS Web Adaptor. Он показывает, был ли настроен ранее данный ArcGIS Web Adaptor. Чтобы узнать тип платформы и номер версии ArcGIS Web Adaptor, щелкните О Web Adaptor на странице конфигурации.

Если вы уже настроили Web Adaptor с порталом ArcGIS Enterprise и хотите обновить портал, чтобы использовать другой Web Adaptor, вам сначала потребуется отменить регистрацию текущего Web Adaptor. Вы можете отменить регистрацию только с Portal Directory; функциональность отмены регистрации недоступна со страницы конфигурации или веб-сайта портала. Подробные инструкции см. в разделе Отмена регистрации ArcGIS Web Adaptor для Portal for ArcGIS.

Для получения подробных инструкций по настройке ArcGIS Web Adaptor выполните шаги, перечисленные ниже.

Внимание:

Перед тем, как продолжить, убедитесь, что ArcGIS Web Adaptor установлен на веб-сайте, работающем через порт 80 или 443. ArcGIS Web Adaptor нельзя настроить с Portal for ArcGIS, если он установлен на сайте, который работает через порт, отличном от 80 и 443.

Настройка ArcGIS Web Adaptor с веб-страницы конфигурации

Для настройки ArcGIS Web Adaptor с веб-страницы конфигурации, которая устанавливается вместе с ним, выполните следующие шаги.

Шаги:
  1. Включите HTTPS для порта 443 на вашем веб-сервере, если вы этого еще не сделали. Подробнее см. раздел Включение HTTPS на веб-сервере.
  2. Откройте страницу настройки Web Adaptor в веб-браузере по протоколу HTTPS, например, https://webadaptorhost.domain.com/webadaptorname/webadaptor.

    Если с порталом будет использоваться псевдоним DNS, то Web Adaptor должен настраиваться на этот псевдоним с использованием URL типа https://<dnsalias.domain.com>/<webadaptorname>/webadaptor.

  3. Выберите Portal for ArcGIS и щелкните Далее.
  4. В качестве URL портала введите URL компьютера, содержащего Portal for ArcGIS. Укажите в URL полное доменное имя компьютера, например, https://portal.domain.com:7443.
  5. Введите имя пользователя и пароль для учетной записи, которая имеет права администратора в Portal for ArcGIS. Обычно используется имя пользователя и пароль основной учетной записи администратора, которая задается при создании портала. Если вы ограничили права или удалили основную учетную запись администратора, необходимо указать учетную запись с административным доступом к веб-сайту портала.
  6. Щелкните Конфигурировать.

С этими настройками ArcGIS Web Adaptor можно использовать на компьютере с Portal for ArcGIS. Теперь у вас есть доступ к порталу через ArcGIS Web Adaptor URL, а не через порт 7443, например: Адрес URL имеет вид: https://webadaptorhost.domain.com/webadaptorname/home.

Если вы настроили ваш портал на использование SSL для передачи данных, обновите URL ярлыков веб-сайта портала и справки с учетом использования протокола https вместо http, иначе вы увидите сообщения об ошибке в браузере при попытке перейти по старым URL-ссылкам ярлыков.

Если планируется настроить Portal for ArcGIS для работы с обратным прокси-сервером вашей организации, потребуется ввести в портал необходимые данные о прокси-сервере. Подробные инструкции см. в разделе Настройка обратного прокси-сервера для работы с порталом.

Настройка ArcGIS Web Adaptor из командной строки

Вы можете настроить ArcGIS Web Adaptor (IIS), используя утилиту командной строки (ConfigureWebAdaptor.exe) в C:\Program Files (x86)\Common Files\ArcGIS\WebAdaptor\IIS\Tools.

Доступны следующие параметры:

ConfigureWebAdaptor.exe /m <Mode> /w <WebAdaptorURL> /g <URL> /u <Username> /p <Password>

  • <Mode> – режим продукта, в котором будет настраиваться ArcGIS Web Adaptor. По умолчанию значением этого параметра является server. Это означает, что ArcGIS Web Adaptor будет настроен для работы с ArcGIS Server. Если вы настраиваете ArcGIS Web Adaptor для работы с Portal for ArcGIS, этот параметр должен иметь значение portal.

  • <WebAdaptorURL> – URL-адрес ArcGIS Web Adaptor, который вы хотите настроить из командной строки. Этот тот же URL-адрес, который используется при доступе к странице настройки Web Adaptor с помощью веб-браузера. Вам требуется указать для приложения ArcGIS Web Adaptor URL-ссылку в формате HTTPS, например, https://webadaptorhost.domain.com/webadaptorname/webadaptor. Если с порталом будет использоваться псевдоним DNS, то Web Adaptor должен настраиваться на этот псевдоним с использованием URL типа https://<dnsalias.domain.com>/<webadaptorname>/webadaptor.

  • <URL> – URL компьютера с Portal for ArcGIS. Укажите в URL полное доменное имя компьютера, например, https://portal.domain.com:7443.

  • <Username> – имя пользователя учетной записи, которая имеет права администратора на Portal for ArcGIS. Обычно используется основная учетная запись администратора, которая задается при создании портала.

  • <Password> – пароль учетной записи, которая имеет права администратора на Portal for ArcGIS. Обычно используется основная учетная запись администратора, которая задается при создании портала.

Пример: ConfigureWebAdaptor.exe /m portal /w https://webadaptorhost.domain.com/webadaptorname/webadaptor /g https://portal.domain.com:7443 /u initialadmin /p secret

После выполнения этой команды ArcGIS Web Adaptor будет настроен на работу с компьютером, на котором установлен Portal for ArcGIS. Теперь у вас есть доступ к порталу через ArcGIS Web Adaptor URL, а не через порт 7443, например: Адрес URL имеет вид: https://webadaptorhost.domain.com/webadaptorname/home.

Если планируется настроить Portal for ArcGIS на работу с прокси-сервером вашей организации, потребуется ввести в портал необходимые данные о прокси-сервере. Подробные инструкции см. в разделе Настройка прокси-сервера для работы с порталом.

Установка ArcGIS Web Adaptor в автоматическом режиме

ArcGIS Web Adaptor может быть установлен путем запуска программы установки с использованием параметров командной строки Windows Installer, как описано ниже. Кроме того, вы можете выполнить установку из пользовательского интерфейса, используя Setup.exe.

Требования установки

Версия ArcGIS Web Adaptor должна всегда совпадать с версией используемого вами Portal for ArcGIS.

На одном компьютере могут быть установлены версия ArcGIS Web Adaptor 10.2.1 или выше, и вместе с ней более ранние версии ArcGIS Web Adaptor. Например, на одном и том же веб-сервере могут быть одновременно установлены версии 10.1 SP1 и 10.6 для ArcGIS Web Adaptor.

Каждый Web Adaptor должен иметь собственное уникальное имя; на одном веб-сервере не должно быть два Web Adaptor с одинаковыми именами. Если во время установки обнаружен конфликт имён, появляется предупреждение. Вам будет необходимо удалить более раннюю версию с тем же именем, чтобы разрешить этот конфликт. Вы также можете изменить имя одного из экземпляров Web Adaptor, чтобы продолжить процесс установки.

Для этого требуются службы Microsoft Internet Information Services (IIS). В случае автоматической инсталляции, если не найдены IIS, установка ArcGIS Web Adaptor выполнена не будет. Перед установкой ArcGIS Web Adaptor в автоматическом режиме должен быть установлен IIS и все необходимые компоненты.

Требуется Microsoft .NET Framework 4.5 или более новая версия. Установка не будет продолжаться, если не найден Microsoft .NET Framework 4.5 или более новая версия.

Прежние версии:

В предыдущих версиях, чтобы установить ArcGIS Web Adaptor, требовалась установка Microsoft .NET Framework 3.5 Service Pack 1 (SP1), 4.0 или 4.5.

По окончании установки ArcGIS Web Adaptor должна быть выполнена его настройка. Дополнительные сведения см. в разделе Настройка ArcGIS Web Adaptor.

Обновите предыдущую версию ArcGIS Web Adaptor

Для обновления предыдущей версии ArcGIS Web Adaptor до 10.6, необходимо удалить предыдущую версию и запустить программу установки для версии 10.6. Во время установки 10.6 вы укажете такое же имя, как и у вашего предыдущего Web Adaptor. Это гарантирует, что URL-адрес вашего Web Adaptor останется таким же, какой он был у предыдущей версии.

Если во время установки 10.6 обнаружен конфликт имён, появляется предупреждение. Вам будет необходимо удалить раннюю версию с тем же именем, чтобы выполнить обновление.

После установки 10.6 настроить ArcGIS Web Adaptor с Portal for ArcGIS.

Прежние версии:

В версии 10.2 и более ранних версиях запуск программы установки обновлял все существующие экземпляры Web Adaptor до самой последней версии.

Параметры командной строки, позволяющие указать веб-сайт и имя для ArcGIS Web Adaptor

Для настройки IIS используются следующие параметры командной строки. Если эти параметры командной строки не использовались в автоматическом режиме установки, то веб-приложение ArcGIS Web Adaptor создается на веб-сайте по умолчанию с использованием стандартного имени arcgis.

  • WEBSITE_ID=<числовое значение веб-сайта>

    Параметр WEBSITE_ID используется для задания веб-сайта, на котором будет создано веб-приложение ArcGIS Web Adaptor. Значением для WEBSITE_ID является число, определенное для каждого веб-сайта на вашем сервере, и назначенное с помощью IIS. Идентификаторы веб-сайтов вашего сервера могут быть найдены в IIS Manager. У сайта по умолчанию идентификатор ID будет 1. По умолчанию параметр WEBSITE_ID задается для веб-сайта по умолчанию (даже если существует несколько веб-сайтов). Параметр WEBSITE_ID чувствителен к регистру.

  • PORT=<номер порта>

    Кроме того, если ваш веб-сайт имеет множество портов, используйте параметр PORT для установки выбранного порта с WEBSITE_ID.

    Например, путем является <path to ArcGIS Web Adaptor (IIS) setup download>\setup.exe /qb VDIRNAME=arcgis_external WEBSITE_ID=4059640 PORT=82.

  • VDIRNAME=<имя ArcGIS Web Adaptor>

    Это необязательный параметр командной строки. По умолчанию свойство VDIRNAME имеет значение arcgis. Если вы не использовали параметр VDIRNAME в командной строке, Web Adaptor будет создан с именем arcgis. Параметр VDIRNAME чувствителен к регистру. Имя Web Adaptor не должно содержать пробелов.

    Например, путем является <path to ArcGIS Web Adaptor (IIS) setup download>\setup.exe /qb VDIRNAME=arcgis_external WEBSITE_ID=4059640.

  • CONFIGUREIIS=TRUE

    Этот дополнительный параметр позволяет программе установки автоматически настраивать и устанавливать все отсутствующие компоненты Microsoft IIS, необходимые для ArcGIS Web Adaptor.

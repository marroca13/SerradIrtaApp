Konfigurieren von mehreren ArcGIS Web Adaptor-Instanzen

Für die Konfiguration eines Portals mit hoher Verfügbarkeit, das die Authentifizierung auf Webebene verwendet, sind mehrere ArcGIS Web Adaptors erforderlich. Um mehrere Web Adaptors mit Portal for ArcGIS zu konfigurieren, muss ein einzelner Endpunkt für den Zugriff auf das Portal verwendet werden. Diese URL muss in der Eigenschaft WebContextURL definiert sein. Mehrere Web Adaptors können dann durch Ausführung der nachfolgenden Schritte konfiguriert werden.

Schritte:
  1. Konfigurieren Sie den ersten ArcGIS Web Adaptor mit Ihrem Portal.
  2. Fügen Sie Ihrem Reverseproxyserver Portal for ArcGIS hinzu. Stellen Sie sicher, dass die Eigenschaft WebContextURL wie folgt festgelegt ist:
    1. Öffnen Sie einen Webbrowser, und melden Sie sich als Administrator Ihrer Organisation beim ArcGIS-Portalverzeichnis an. Die URL hat das Format https://portal.domain.com:7443/arcgis/portaladmin.
    2. Klicken Sie auf System > Properties > Update Properties. Dadurch wird der Portal for ArcGIS-Service neu gestartet. Sie müssen warten, bis der Neustart abgeschlossen wurde, bevor Sie den Vorgang fortsetzen können.
    3. Geben Sie im Dialogfeld Systemeigenschaften aktualisieren den folgenden JSON-Code ein, und geben Sie dabei die URL Ihres eigenen Reverseproxyserver oder DNS-Alias ein, wie sie Benutzern außerhalb der Firewall der Organisation angezeigt werden.
      {
   "WebContextURL": "https://reverseproxy.domain.com/myorg"
}
    4. Klicken Sie auf Update Properties.
  3. Konfigurieren Sie weitere Web Adaptor.

Alle mit dem Portal konfigurierten Web Adaptor werden auf der Seite mit der Zusammenfassung der Konfiguration aufgeführt. Benutzer greifen über den Reverseproxyserver- oder die DNA-Alias-URL auf das Portal zu.

Konfigurieren von ArcGIS Web Adaptor

Nachdem Sie ArcGIS Web Adaptor installiert haben, müssen Sie ihn für die Verwendung mit einer ArcGIS Server-Site konfigurieren. Dazu verwenden Sie eine mit dem Web Adaptor installierte Konfigurations-Webseite bzw. ein Befehlszeilenprogramm. Als Sicherheitsmaßnahme kann der Zugriff auf die Konfigurationsseite und das Befehlszeilenprogramm nur über den Computer erfolgen, auf dem der Web Adaptor gehostet wird.

Wenn Sie die Konfigurationsseite öffnen, wird der aktuelle Status des Web Adaptor angezeigt. Dies gibt an, ob der Web Adaptor zuvor konfiguriert wurde. Um den Plattformtyp und die Versionsnummer des Web Adaptor anzuzeigen, klicken Sie auf der Konfigurationsseite auf Info.

Vollständige Anweisungen zum Konfigurieren des Web Adaptor finden Sie im folgenden Abschnitt.

Konfigurieren des Web Adaptor über die Konfigurations-Webseite

Um den Web Adaptor über die Konfigurations-Website zu konfigurieren, die mit der Software installiert wurde, führen Sie die folgenden Schritte aus.

Schritte:
  1. Öffnen Sie die Konfigurationsseite des Web Adaptor in einem Webbrowser. Die URL hat das folgende Format: http://webadaptorhost.domain.com/webadaptorname/webadaptor.
  2. Wählen Sie ArcGIS-Server aus, und klicken Sie auf Weiter.
  3. Geben Sie für ArcGIS-Server-URL die URL zu einem der ArcGIS Server-Computer Ihrer ArcGIS Server-Site ein. Diese URL wird verwendet, um alle Computer der Site zu ermitteln und sie beim Web Adaptor zu registrieren. Die URL hat beispielsweise das Format http://gisserver.domain.com:6080.
  4. Geben Sie einen Benutzernamen und ein Kennwort für ein Konto ein, dem Administratorberechtigungen für Ihre ArcGIS Server-Site zugewiesen wurden. In der Regel geben Sie den bei der Erstellung der Site festgelegten Benutzernamen und das entsprechende Kennwort für das primäre Site-Administratorkonto an. Falls Sie das primäre Site-Administratorkonto deaktiviert haben, müssen Sie ein Konto mit administrativem Zugriff auf die Site angeben.
  5. Legen Sie fest, ob Benutzer die Site über den Web Adaptor verwalten dürfen. Die Administration der Site über Web Adaptor ist standardmäßig deaktiviert. Dies ist die empfohlene Option. Hier finden Sie einige Hinweise zu dieser Option:
    • Wenn der administrative Zugriff deaktiviert wird, ist der Zugriff auf ArcGIS Server Manager und das ArcGIS Server-Administratorverzeichnis über den Web Adaptor für externe Benutzer gesperrt. Außerdem werden Benutzer von ArcGIS Desktop am Herstellen von Administrator- oder Publisher-Verbindungen mit ArcGIS Server gehindert. Beachten Sie, dass Sie, unabhängig davon, ob diese Option aktiviert oder deaktiviert ist, weiterhin die Möglichkeit haben, Benutzerverbindungen zwischen ArcGIS Desktop und ArcGIS Server herzustellen.
    • Wenn der administrative Zugriff über den Web Adaptor deaktiviert ist, können Sie in ArcGIS Desktop auf Manager und das Administratorverzeichnis zugreifen sowie eine Verbindung zum Server herstellen, indem Sie eine direkte URL zu einem der ArcGIS Server-Computer der Site verwenden. Beispiel:
      • Für den Zugriff auf Manager müssen Sie das URL-Format http://gisserver.domain.com:6080/arcgis/manager verwenden.
      • Um auf das Administratorverzeichnis zugreifen zu können, müssen Sie das URL-Format http://gisserver.domain.com:6080/arcgis/admin verwenden.
      • Zum Herstellen einer Serververbindung in ArcGIS Desktop verwenden Sie das URL-Format http://gisserver.domain.com:6080/arcgis.
    • Wenn ArcGIS Server mit der Authentifizierung auf Webebene konfiguriert wurde, müssen Sie die Administration über den Web Adaptor aktivieren. Dadurch können Benutzer mit Publisher- und Administratorberechtigungen in Ihrem Enterprise-Identitätsspeicher Services aus ArcGIS Desktop veröffentlichen. Wenn die Benutzer in diesen Rollen eine Verbindung mit dem Server in ArcGIS Desktop herstellen, müssen sie die Web Adaptor-URL angeben.
  6. Klicken Sie auf Konfigurieren, um die Einstellungen auf den Web Adaptor anzuwenden.

Wenn die Konfiguration erfolgreich war, wird am unteren Rand der Konfigurationsseite (grün hervorgehoben) eine Liste der vom Web Adaptor erkannten ArcGIS Server-Computer angezeigt. Sie können jederzeit zu dieser Seite zurückkehren, um die Liste erkannter Computer anzuzeigen und die Web Adaptor-Einstellungen anzupassen.

Konfigurieren des Web Adaptor über die Befehlszeile

Der Web Adaptor kann mit dem Befehlszeilendienstprogramm (ConfigureWebAdaptor.exe) unter C:\Program Files (x86)\Common Files\ArcGIS\WebAdaptor\IIS\Tools konfiguriert werden.

Folgende Parameter stehen zur Verfügung:

ConfigureWebAdaptor.exe/m <Modus>/w <Web-Adaptor-URL>/g <URL> /u <Benutzername> /p <Kennwort> /a <Administratorzugriff aktiviert>

  • <Modus>: Der Produktmodus, in dem der Web Adaptor konfiguriert wird. Dieser Parameter ist standardmäßig auf Server festgelegt, was bedeutet, dass der Web Adaptor für die Verwendung mit ArcGIS Server konfiguriert wird. Wenn Sie den Web Adaptor für die Verwendung mit Portal for ArcGIS konfigurieren, geben Sie diesen Parameter als Portal ein.

  • <Web Adaptor-URL>: Die URL des Web Adaptor, der über die Befehlszeile konfiguriert werden soll. Dies ist dieselbe URL, die Sie verwenden, als wenn Sie die Konfigurationsseite des Web Adaptor in einem Webbrowser aufrufen. Wenn Web Adaptor an einem anderen Port als dem Standardport 80 installiert wurde, müssen Sie den Port in der URL angeben.

  • <URL>: Die URL zu einem der ArcGIS Server-Computer Ihrer ArcGIS Server-Site. Diese URL wird verwendet, um alle Computer der Site zu ermitteln und sie beim Web Adaptor zu registrieren.

  • <Benutzername>: Der Benutzername eines Kontos, dem Administratorberechtigungen für die ArcGIS Server-Site zugewiesen wurden. In der Regel geben Sie das bei der Erstellung der Site festgelegte primäre Site-Administratorkonto an.

  • <Kennwort>: Das Kennwort eines Kontos, dem Administratorberechtigungen für die ArcGIS Server-Site zugewiesen wurden. In der Regel geben Sie das bei der Erstellung der Site festgelegte primäre Site-Administratorkonto an.

  • <AdminAccessEnabled>: Gibt an, ob ArcGIS Server Manager und das ArcGIS Server-Administratorverzeichnis über den Web Adaptor verfügbar sind. Als Sicherheitsmaßnahme ist dieser Parameter standardmäßig auf False festgelegt. Um den administrativen Zugriff zu aktivieren, legen Sie diesen Parameter auf True fest.

Beispiel: ConfigureWebAdaptor.exe /m server /w http://webadaptorhost.domain.com/webadaptorname/webadaptor /g http://gisserver.domain.com:6080 /u siteadmin /p secret /a false

Ältere Versionen:

In Version 10.2 und vorherigen Versionen musste die Zeichenfolge /server an das Ende der Web Adaptor-URL angefügt werden, beispielsweise /w http://webadaptorhost.domain.com/webadaptorname/webadaptor/server. In 10.2.1 und späteren Versionen ist dies nicht mehr erforderlich.

Nach dem Konfigurieren des Web Adaptor

Nachdem der Web Adaptor zur Verwendung konfiguriert wurde, müssen einige wichtige Punkte beachtet werden:

Hinzufügen oder Entfernen von ArcGIS Server-Computern

Wenn Sie der Site ArcGIS Server-Computer hinzufügen bzw. aus ihr entfernen, dauert es eine Minute, bis der Web Adaptor die Änderungen an der Site erkennt. Falls der Web Adaptor die hinzugefügten bzw. entfernten Computer sofort erkennen soll, können Sie ihn unter Befolgung der oben aufgeführten Anweisungen entsprechend neu konfigurieren.

Installieren mehrerer Web Adaptors

Sie können verschiedene Web-Einstiegspunkte in Ihrer ArcGIS Server-Site bereitstellen, indem Sie mehrere Web Adaptor-Instanzen installieren. Dies kann hilfreich sein, wenn Sie die Site für Clients verfügbarer machen, Unterstützung für ältere Anwendungen bereitstellen oder Zugriff auf Veröffentlichungs- und Verwaltungsfunktionen für bestimmte Benutzer in einer umfassend geschützten Umgebung zulassen möchten. Weitere Informationen finden Sie unter Bereitstellungsszenarien.

Auf einem Computer können mehrere Web Adaptor-Instanzen gleichzeitig installiert werden oder die Installation kann über mehrere getrennte Computer verteilt werden. Wenn Sie mehrere Web Adaptor-Instanzen auf derselben Website (Port) installieren möchten, müssen diese verschiedene Namen verwenden. Beispielsweise können zwei Web Adaptor-Instanzen mit dem Namen arcgis nicht auf derselben Website vorhanden sein. Wenn Sie mehrere Web Adaptor-Instanzen mit demselben Namen installieren möchten, müssen diese auf getrennten Websites (Ports) installiert werden.

Aktivieren von HTTPS für ArcGIS Server

Standardmäßig verwendet ArcGIS Server HTTP und HTTPS für die Kommunikation. Wenn Sie das Kommunikationsprotokoll von ArcGIS Server auf "Nur HTTP" oder "Nur HTTPS" aktualisieren, dauert es eine Minute, bis der Web Adaptor die Änderungen an der Site erkennt.

Ältere Versionen:

In Version 10.2.1 und vorherigen Versionen musste ArcGIS Web Adaptor nach dem Aktualisieren des ArcGIS Server-Kommunikationsprotokolls neu konfiguriert werden. In 10.2.2 und späteren Versionen ist dies nicht mehr erforderlich.

Weitere Informationen zum Kommunikationsprotokoll von ArcGIS Server finden Sie unter Sichern der Kommunikation mit ArcGIS Server.

Vollständige Anweisungen zum Installieren mehrerer Web Adaptor-Instanzen finden Sie unter Installieren mehrerer Web Adaptor-Instanzen.

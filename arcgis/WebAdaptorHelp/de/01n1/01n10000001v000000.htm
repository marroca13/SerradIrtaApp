Automatisches Installieren von ArcGIS Web Adaptor

Die Installation von ArcGIS Web Adaptor kann, wie nachfolgend beschrieben, durch die Ausführung des Installationsprogramms mit Befehlszeilenparametern für Windows Installer erfolgen. Sie haben auch die Möglichkeit, die Installation über die Benutzeroberfläche durchzuführen, indem Sie Setup.exe verwenden.

Installationsvoraussetzungen

Die Version von ArcGIS Web Adaptor muss immer mit der verwendeten Portal for ArcGIS-Version übereinstimmen.

Die Version 10.2.1 oder eine höhere Version von ArcGIS Web Adaptor kann zusammen mit einer früheren Version von ArcGIS Web Adaptor auf dem gleichen Computer installiert sein. Beispielsweise können die Versionen 10.1 SP1 und 10.6 von ArcGIS Web Adaptor auf dem gleichen Webserver installiert sein.

Jeder Web Adaptor muss einen eindeutigen Namen haben; es können keine zwei Web Adaptor mit dem gleichen Namen auf einem Webserver installiert werden. Wenn ein Namenskonflikt während der Installation erkannt wird, wird eine Warnmeldung angezeigt. Sie müssen die frühere Version mit dem gleichen Namen deinstallieren, um den Konflikt zu lösen. Alternativ können Sie den Namen eines Web Adaptors ändern, um mit der Installation fortzufahren.

Microsoft Internet Information Services (IIS) ist erforderlich. Folgendes gilt ausschließlich für die automatische Installation: Wenn IIS nicht gefunden wird, wird die Installation von ArcGIS Web Adaptor nicht fortgesetzt. IIS und alle erforderlichen Komponenten müssen vor der automatischen Installation von ArcGIS Web Adaptor installiert werden.

Microsoft .NET Framework 4.5 oder höher ist erforderlich. Wenn Microsoft .NET Framework 4.5 oder höher nicht gefunden wird, wird die Installation nicht durchgeführt.

Ältere Versionen:

In früheren Versionen waren für die Installation von ArcGIS Web Adaptor Microsoft .NET Framework 3.5 Service Pack 1 (SP1), 4.0 oder 4.5 erforderlich.

ArcGIS Web Adaptor muss nach der Installation konfiguriert werden. Weitere Informationen finden Sie unter Konfigurieren von ArcGIS Web Adaptor.

Upgrade einer früheren Version von ArcGIS Web Adaptor

Um ein Upgrade einer früheren ArcGIS Web Adaptor-Version auf 10.6 durchzuführen, müssen Sie die frühere Version deinstallieren und das 10.6-Setup installieren. Geben Sie bei der 10.6-Installation den Namen der früheren Web Adaptor-Version an. Dadurch wird sichergestellt, dass die Web Adaptor-URL mit der der früheren Version identisch ist.

Wenn ein Namenskonflikt während der Installation von 10.6 erkannt wird, wird eine Warnmeldung angezeigt. Sie müssen die frühere Version mit dem gleichen Namen deinstallieren, um das Upgrade durchzuführen.

Nachdem Sie 10.6 installiert haben, müssen Sie ArcGIS Web Adaptor mit Portal for ArcGIS konfigurieren.

Ältere Versionen:

In Versionen bis 10.2 wurde für alle vorhandenen Web Adaptors durch die Ausführung des Setup-Programms ein Upgrade auf die neueste Version durchgeführt.

Befehlszeilenparameter zum Angeben der Website und des Namens für ArcGIS Web Adaptor

Für die Konfiguration von IIS werden die folgenden Befehlszeilen verwendet. Wenn diese Befehlszeilenparameter während einer automatischen Installation nicht aufgerufen werden, wird die ArcGIS Web Adaptor-Webanwendung auf der Standard-Website mit dem Standardnamen arcgis installiert.

  • WEBSITE_ID=<numerischer Wert der Website>

    Der Parameter "WEBSITE_ID" wird verwendet, um die Website anzugeben, auf der die ArcGIS Web Adaptor-Webanwendung erstellt wird. Der Wert von "WEBSITE_ID" ist eine Zahl, die für jede Website auf dem Server spezifisch ist und vom IIS zugewiesen wird. Die IDs für die Websites auf dem Server finden Sie im IIS-Manager. Die Standardwebsite hat eine Website-ID von 1. Standardmäßig ist der Parameter "WEBSITE_ID" auf die Standard-Website festgelegt (auch wenn mehrere Websites vorhanden sind). Beim Parameter "WEBSITE_ID" wird die Groß-/Kleinschreibung beachtet.

  • PORT=<Portnummer>

    Wenn Sie darüber hinaus über eine Website mit mehreren Ports verfügen, sollten Sie den Parameter "PORT" verwenden, um die Installation am angegebenen Port der "WEBSITE_ID" durchzuführen.

    Der Pfad lautet beispielsweise <path to ArcGIS Web Adaptor (IIS) setup download>\setup.exe /qb VDIRNAME=arcgis_external WEBSITE_ID=4059640 PORT=82.

  • VDIRNAME=<Name von ArcGIS Web Adaptor>

    Dieser Befehlszeilenparameter ist optional. Standardmäßig ist die Eigenschaft "VDIRNAME" auf arcgis festgelegt. Wenn Sie den Parameter "VDIRNAME" nicht in der Befehlszeile aufrufen, wird ArcGIS Web Adaptor als arcgis erstellt. Beim Parameter "VDIRNAME" wird Groß- und Kleinschreibung unterschieden. Der Web Adaptor-Name darf keine Leerzeichen enthalten.

    Der Pfad lautet beispielsweise <path to ArcGIS Web Adaptor (IIS) setup download>\setup.exe /qb VDIRNAME=arcgis_external WEBSITE_ID=4059640.

  • CONFIGUREIIS=TRUE

    Mit diesem optionalen Parameter kann das Installationsprogramm alle fehlenden Microsoft IIS-Komponenten, die für ArcGIS Web Adaptor erforderlich sind, konfigurieren und installieren.

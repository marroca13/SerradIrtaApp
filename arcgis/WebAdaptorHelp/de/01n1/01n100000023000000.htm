Info zu ArcGIS Web Adaptor

ArcGIS Web Adaptor (IIS) ist eine Komponente von Portal for ArcGIS, die die Integration Ihres Portals in den vorhandenen Webserver und die Sicherheitsmechanismen Ihrer Organisation ermöglicht. ArcGIS Web Adaptor ist für die Verwendung der integrierten Windows-Authentifizierung mit Ihrem Portal erforderlich.

Der Web Adaptor wird als Anwendung auf der vorhandenen Website ausgeführt und leitet Anfragen an den Computer weiter, auf dem Portal for ArcGIS gehostet wird. Das IIS-Setup ist mit IIS in Windows 7, 8, 8.1, 10, Server 2008, 2008 R2, Server 2012 und 2012 R2 kompatibel. Dies entspricht den IIS-Versionen 7.0, 7.5, 8.0, 8.5 und 10.

Vorteile des Web Adaptor

Der Web Adaptor bietet die folgenden Vorteile:

  • Möglichkeit zur Integration von Portal for ArcGIS in den vorhandenen Webserver Ihrer Organisation. Durch Einbeziehung eines Webservers in die Site erhalten Sie die Möglichkeit zum Hosten von Webanwendungen, die Ihre GIS-Services verwenden.

  • Ermöglicht Ihnen das Verwenden des Identitätsspeichers der Organisation und der Sicherheitsrichtlinien auf Webebene. Wenn Sie beispielsweise IIS verwenden, kann über die integrierte Windows-Authentifizierung eingeschränkt werden, wer auf das Portal zugreifen kann.Sie können auch eine Public Key-Infrastruktur (PKI) oder einen anderen Identitätsspeicher verwenden, für den der Webserver über eine integrierte oder erweiterbare Unterstützung verfügt. Dadurch können Sie Single Sign-On oder eine andere benutzerdefinierte Authentifizierung beim Anmelden für die Verwendung von Services, Anwendungen und Portal for ArcGIS bereitstellen.

  • Ermöglicht die Verfügbarkeit von Portal for ArcGIS über einen anderen Site-Namen als den Standardnamen arcgis.

  • Ermöglicht die Verfügbarkeit von Portal for ArcGIS über Port 80 oder 443.

    Hinweis:

    Sie können den Web Adaptor nur mit Port 80 oder 443 verwenden. Die Verwendung anderer Ports wird nicht unterstützt.

Szenarien für die Web Adaptor-Bereitstellung

Die Version des Web Adaptor muss immer mit der Version von Portal for ArcGIS übereinstimmen. Wenn Sie mehrere Web Adaptors auf einem Webserver konfiguriert haben, können Sie verschiedene Versionsnummern für alle Web Adaptors verwenden.

Ältere Versionen:

In Version 10.2 und früher mussten alle Web Adaptors auf einem Webserver dieselbe Versionsnummer verwenden. Durch Ausführen des Setup-Programms wurde für alle vorhandenen Web Adaptors ein Upgrade auf die aktuelle Version durchgeführt.

Der Web Adaptor ist plattformunabhängig von Portal for ArcGIS. Daher muss der bereitgestellte Web Adaptor nicht mit der Betriebssystemplattform des Portals übereinstimmen. Wenn das Portal beispielsweise unter Linux ausgeführt wird, können Sie ArcGIS Web Adaptor (IIS) für die Verwendung mit Portal for ArcGIS bereitstellen.

Zugreifen auf das Portal bei installiertem Web Adaptor

Bei der Installation und Konfiguration des Web Adaptor hat die URL, über die Sie auf das Portal zugreifen, das folgende Format: https://webadaptorhost.domain.com/webadaptorname/home. Wenn beispielsweise der Computer, auf dem der Web Adaptor gehostet wird, den Namen wa mit der Domäne myorg.net und Ihr Web Adaptor den Namen arcgis hat, rufen Sie das Portal über die URL https://wa.myorg.net/arcgis/home auf.

Wenn Sie die Verwendung von HTTPS für die gesamte Kommunikation im Portal konfiguriert haben, sollten Sie die installierte Portal-Website und die Verknüpfungs-URLs der Hilfe zur Verwendung von https anstelle von http aktualisieren. Andernfalls wird im Browser ein Fehler angezeigt, wenn Sie versuchen, auf die ursprünglichen Verknüpfungs-URLs zuzugreifen.

Das Setup-Verfahren des Web Adaptor

Der Web Adaptor verfügt über ein eigenes Installationsprogramm und eigene Installationsanweisungen, getrennt von der Portal for ArcGIS-Installation. Sie müssen den Web Adaptor auf einem Computer mit einem Webserver installieren. Dabei kann es sich um einen Computer handeln, auf dem bereits Portal for ArcGIS ausgeführt wird, oder um einen separaten Computer.

Weitere Informationen zum Setup-Verfahren finden Sie unter Überblick über die Implementierung von Web Adaptor mit Portal for ArcGIS.

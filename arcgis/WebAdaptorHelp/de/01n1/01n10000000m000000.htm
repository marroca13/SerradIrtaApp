Bereitstellungsszenarien

ArcGIS Server ist skalierbar und für große und kleine Bereitstellungen geeignet. Wenn Sie mit der Zusammenstellung der Site beginnen, können Sie im kleinen Umfang beginnen und alle Komponenten auf einem Computer installieren. Wenn Sie eine Produktions-Site bereitstellen oder mehr Benutzer auf der Site gehandhabt werden müssen, können Sie weitere ArcGIS Server-Computer hinzufügen. Sie können die Site auch in eine vorhandene IT-Infrastruktur integrieren, indem Sie einen eigenen Unternehmens-Webserver (z. B. über ArcGIS Web Adaptor), eine eigene Datenbank oder eigene Enterprise-Identity-Provider verwenden. Darüber hinaus kann auf dem ArcGIS Server-Computer die Unterstützung kritischer Geschäftsoperationen durch Konfigurationen mit hoher Verfügbarkeit eingerichtet werden.

In dieser Hilfe werden unterschiedliche Möglichkeiten zur Erstellung der Architektur einer ArcGIS Server-Site erläutert, die auf unterschiedliche Kapazitäten und Verfügbarkeitsanforderungen zugeschnitten sind. Die folgenden Begriffe werden verwendet, um die einzelnen Bereitstellungsszenarien zu erklären:

  • Site – Eine ArcGIS Server-Site besteht aus mehreren Komponenten, z. B. einem ArcGIS Server-Computer und ArcGIS Web Adaptor. Diese Komponenten können optional auf mehrere Computer verteilt werden, um die Rechenleistung und Redundanz zu erhöhen. Eine detaillierte Beschreibung finden Sie unter Aufbau einer ArcGIS Server-Site.
  • ArcGIS Server – Die Hauptkomponente der Site, die die Anfragen an die GIS-Web-Services erfüllt. Über ArcGIS Server können Karten gezeichnet, Werkzeuge ausgeführt, Bilddaten bereitgestellt und viele andere von ArcGIS bereitgestellte Vorgänge durchgeführt werden.
  • Reverseproxyserver – Eine optionale Komponente eines Drittanbieters in Ihrer Organisation, die zwischen einen Client und einen Server in einer Netzwerkinfrastruktur geschaltet wird. Eingehende Anforderungen werden vom Proxy verarbeitet. Dieser übernimmt die Rolle des Clients in Bezug auf den gewünschten Server oder Service auf dem Server. Die meisten Organisationen richten einen Proxyserver ein, damit die Site nicht direkt für Clients erreichbar ist. Weitere Informationen finden Sie unter Verwenden eines Reverseproxyservers mit ArcGIS Server.
  • ArcGIS Web Adaptor – Eine optionale Komponente, die die Konfiguration eines Web-Einstiegspunktes zu Ihrer Site ermöglicht. Sie wird in den Webserver integriert und verteilt eingehende Anfragen an die ArcGIS Server-Computer. Weitere Informationen finden Sie unter ArcGIS Web Adaptor.
  • Netzwerk-Load-Balancer (NLB) – Eine Drittanbieterkomponente, die eine Lastverteilung des Netzwerkdatenverkehrs anhand eines Verteilungsalgorithmus auf eine Reihe von Hosts vornimmt und damit die Skalierbarkeit und Verfügbarkeit von Webservices verbessert. Darüber hinaus wird damit normalerweise hohe Verfügbarkeit bereitgestellt, da Computerausfälle registriert und der Datenverkehr automatisch auf verfügbare Computer umverteilt wird.
  • Serververzeichnisse – Eine Gruppe von ArcGIS Server-Verzeichnissen, die bestimmte Dateitypen beinhalten, die die Services unterstützen. Diese Dateien beinhalten Karten-Caches, Suchindizes und Ergebnisse von Geoverarbeitungsaufträgen. Weitere Informationen hierzu finden Sie unter Serververzeichnisse.
  • Konfigurationsspeicher – Ein Dateiverzeichnis, das Konfigurationsinformationen zu der Site enthält, z. B. die Liste der ArcGIS Server-Server, die der Site angehören. Der Konfigurationsspeicher muss für den ordnungsgemäßen Betrieb der Site zur Verfügung stehen. Weitere Informationen finden Sie unter Konfigurationsspeicher.
  • Daten – Daten, die Webservices unterstützen, z. B. Feature-Classes, Werkzeuge, Bilddaten und Locators. Weitere Informationen finden Sie unter Bereitstellen der Daten für ArcGIS Server.

Die in den folgenden Szenarien beschriebenen Themen stellen Bereitstellungsrichtlinien dar, die beim Erstellen der ArcGIS Server-Site zu berücksichtigen sind. Sie könnten die Site genauso konfigurieren, wie es in einem dieser Szenarien dargestellt wird, diese Konfigurationen sind jedoch flexibel und können an Ihre speziellen Anforderungen und Hardwareressourcen angepasst werden.

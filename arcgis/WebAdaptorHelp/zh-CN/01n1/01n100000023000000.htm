关于 ArcGIS Web Adaptor

ArcGIS Web Adaptor (IIS) 是 Portal for ArcGIS 的组件，可用于将门户与现有 web 服务器以及组织的安全机制相集成。ArcGIS Web Adaptor 是您门户使用 Windows 身份验证的所必需的组件。

Web Adaptor 是一个运行在您现有网站上的应用程序，它能够将请求转发到托管 Portal for ArcGIS 的计算机中。IIS 安装程序与 Windows 7、8、8.1、10、Server 2008、2008 R2、Server 2012 和 2012 R2 中的 IIS 兼容。此安装程序与 7.0、7.5、8.0、8.5 和 10 版本的 IIS 相对应。

Web Adaptor 的优势

Web Adaptor 具有以下优势：

  • 允许将 Portal for ArcGIS 与组织内现有的 Web 服务器相集成。通过向站点中增加 web 服务器，就可以获得承载 Web 应用程序的能力，这些 Web 应用程序可使用自己发布的 GIS 服务。

  • 使您能够在 Web 层级别使用组织的标识存储和安全策略。例如，如果使用 IIS，则可以使用集成的 Windows 身份验证限制进入门户的用户。如果是具有内置或扩展支持的 Web 服务器，您也可以使用公钥基础设施 (PKI) 或其他标识存储。这允许您在进行登录以使用服务、Web 应用程序和 Portal for ArcGIS 时可以提供单点登录或其他身份验证体验。

  • 使您能够通过站点名称（而非默认的 arcgis）显示 Portal for ArcGIS。

  • 使您能够通过端口 80 或 443 显示 Portal for ArcGIS。

    注：

    只能通过端口 80 或 443 来使用 Web Adaptor。不支持使用其他不同的端口。

Web Adaptor 部署方案

Web Adaptor 的版本必须始终与 Portal for ArcGIS 的版本相匹配。如果您已经在单个 Web 服务器上配置了多个 Web Adaptor，则可将不同的版本号用于所有的 Web Adaptor。

旧版本：

在 10.2 或更低版本中，单个 Web 服务器上的所有 Web Adaptor 都需要使用相同的版本号；运行安装程序会将所有的现有 Web Adaptor 升级为最新版本。

Web Adaptor 是独立于 Portal for ArcGIS 的平台；因此，部署的 Web Adaptor 不必与门户的操作系统平台相匹配。例如，如果您的门户在 Linux 中运行，那么您可以部署 ArcGIS Web Adaptor (IIS) 来使用 Portal for ArcGIS。

安装了 Web Adaptor 的情况下对门户的访问

安装并配置 Web Adaptor 后，用于访问门户的 URL 格式为 https://webadaptorhost.domain.com/webadaptorname/home。例如，如果托管 Web Adaptor 的计算机名为 wa，域为 myorg.net，且 Web Adaptor 名为 arcgis，您将使用 URL https://wa.myorg.net/arcgis/home 来访问门户。

如果已将门户配置为在所有通信中使用 HTTPS，则应将安装的门户网站和帮助快捷方式 URL 更新为使用 https 而不是 http；否则在尝试访问原始快捷方式 URL 时浏览器会显示失败。

Web Adaptor 安装体验

Web Adaptor 有其自己的安装程序和安装指南，独立于 Portal for ArcGIS 安装程序。您必须将 Web Adaptor 安装在一个运行 Web 服务器的计算机上。它可以是已经运行 Portal for ArcGIS 的计算机，也可以是一台独立的计算机。

有关安装体验的详细信息，请参阅使用 Portal for ArcGIS 执行 Web Adaptor 概述。

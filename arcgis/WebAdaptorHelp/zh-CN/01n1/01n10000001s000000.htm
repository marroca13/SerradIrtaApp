ArcGIS 中的门户角色

Portal for ArcGIS 在 ArcGIS 系统内部信息的组织和共享方面起着核心作用。门户为地图和应用程序提供易于操作的、可搜索的资料档案库。它还有助于创建和共享新地图和应用程序。

这些地图和应用程序中丰富的信息均来自组织内部的数据库。此数据库中的 GIS 内容通过由 ArcGIS Server 托管的 web 服务共享给其他设备。门户帮助地图和应用程序的创建者查找和使用这些 web 服务。门户提供访问 GIS 内容的简易窗口，而无需 GIS 软件培训。

一些组织发布一整套 web 服务，可用于通过专有数据创建完整的自主应用程序。但是，在很多情况下您可能希望使用详细的底图、地点查找器、地址查找器和 ArcGIS Online 上的其他 GIS web 服务来补充自己的服务。这种情况下，将自己的 ArcGIS Server 服务与在 Esri 云中托管和不断更新的服务合并。

如果地图和应用程序不能访问 Internet，可以使用 Data Appliance for ArcGIS 来提供相同的底图、地址查找器以及通常可以在 ArcGIS Online 中找到的其他 web 服务。这种部署方式可以在组织防火墙后完全托管。

如果支持内部和外部用户的组合，可以部署两个门户：一个位于防火墙后，而另一个显示于 web。希望完全控制数据位置和更新或数据不允许异地托管时，此方法十分适用。对于面向外部的应用程序和服务，使用组织订阅代替 ArcGIS Online。

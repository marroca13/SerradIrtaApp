部署方案

ArcGIS Server 设计为具有可扩展性，可容纳小型部署和大型部署。首次构建站点时，您可能想要启动小型部署并将所有组件安装在单机上。当您部署生产站点或您的站点需要处理更多用户请求时，可以添加更多 ArcGIS Server 计算机。还可以将站点通过您自己的企业级 web 服务器（例如，通过 ArcGIS Web Adaptor）、数据库或企业级身份提供者集成到现有的 IT 基础架构。还可以通过使用高可用性配置将 ArcGIS Server 配置为支持关键业务操作。

本帮助手册介绍了为满足不同的站点容量和可用性要求而构建 ArcGIS Server 站点的不同方法。以下术语用于解释每种部署方案：

  • 站点 - ArcGIS Server 站点由多个组件构成，如 ArcGIS Server 计算机和 ArcGIS Web Adaptor。可将这些组件有选择地分布到多台计算机上，以提高计算能力和增加冗余。有关更加详细的说明，请参阅 ArcGIS Server 站点内部。
  • ArcGIS Server - 站点的主要组件，用于处理发布到 GIS web 服务的请求。ArcGIS Server 可生成地图、运行工具、提供影像以及执行 ArcGIS 所提供的众多其他操作。
  • 反向代理服务器 - 是组织中可选的第三方组件，放置在网络基础设施中客户端与服务器之间。代理处理传入的请求并代表客户端与所需服务器或服务器上的服务进行交互。大多数组织都安装了反向代理，因此站点不会直接显示给客户端。有关详细信息，请参阅将反向代理服务器与 ArcGIS Server 结合使用。
  • ArcGIS Web Adaptor - 可选组件，用于配置站点的 Web 入口点。ArcGIS Web Adaptor 与 Web 服务器相集成并将收到的请求分配给 ArcGIS Server 计算机。有关详细信息，请参阅关于 ArcGIS Web Adaptor。
  • 网络负载均衡器 (NLB) - 使用分配算法在许多主机之间加载均衡网络流量的第三方组件，用于增强 web 服务的可扩展性和可用性。通常，网络负载均衡器还通过检测计算机故障和向可用计算机自动重新分配流量来实现高可用性。
  • 服务器目录 - 一组包含某些特定类型的文件的 ArcGIS Server 目录，这些文件可对您的服务提供支持。这些文件包括地图缓存、搜索索引和地理处理作业结果。有关详细信息，请参阅关于服务器目录。
  • 配置存储 - 包含站点相关配置信息（例如参与站点的 ArcGIS Server 的列表）的文件目录。该配置存储为必需项以使站点运行。有关详细信息，请参阅关于配置存储。
  • 数据 - 用于支持 Web 服务的数据，例如要素类、工具、影像和定位器。有关详细信息，请参阅使数据可供 ArcGIS Server 访问。

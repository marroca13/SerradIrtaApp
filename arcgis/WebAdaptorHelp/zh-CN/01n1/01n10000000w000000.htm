在 Windows Server 2008/2008 R2 中启用 IIS 及所需 IIS 组件

在 Windows Server 2008/2008 R2 中，ArcGIS Web Adaptor 要求启用 IIS 及特定 IIS 组件。如果未检测到 IIS 或未启用特定的 IIS 组件，则安装程序将无法继续。

如果您已启用 IIS 但缺少所需 IIS 组件，则安装程序将显示一条消息，提示缺少某些 IIS 组件。您可以选择允许安装程序自动启用所需 IIS 组件。但是根据组织安全策略，也可能需要按以下说明手动启用所需的 IIS 组件。

要了解在 Windows Server 2008/2008 R2 中启用 IIS 及所需 IIS 组件的方法，请参阅以下说明。

  1. 单击开始 > 管理工具 > 服务器管理器。

  2. 在服务器管理器 对话框的左侧面板上，单击角色。
    • 如果尚未启用 IIS，请在角色摘要面板上单击添加角色。单击下一步，并启用列表中的 Web 服务器 (IIS)。然后单击下一步，并在左侧面板中选择角色服务。
    • 如果已启用 IIS，但未启用全部所需组件，则在右侧 Web 服务器 (IIS) 面板上单击添加角色服务。

  3. 在选择角色服务 对话框中，确认下列各个 Web 服务器组件是否已经开启。

  4. 启用所需 IIS 组件后，单击下一步 > 安装。

所需 IIS 组件

以下列出的 IIS 组件可以满足运行 Web Adaptor 的最低要求。如果您已启用了其他 IIS 组件，也无需删除它们。

  • Web 服务器
    • 常见 HTTP 功能
      • 静态内容
      • 默认文档
    • 应用程序开发
      • ASP.NET
      • .NET 扩展性
      • ISAPI 扩展
      • ISAPI 筛选器
    • 安全性
      • 基本身份验证
      • Windows 身份验证
      • 请求筛选
  • 管理工具
    • IIS 管理控制台
    • IIS 管理脚本和工具
    • 管理服务
    • IIS 6 管理兼容性
      • IIS 6 元数据库兼容性

配置シナリオ

ArcGIS Server はスケーラブルであり、小規模な配置と大規模な配置の両方に対応できるように設計されています。サイトの構築に着手したときには、小規模なサイトから始めて、1 台のコンピューターにすべてのコンポーネントをインストールすることができます。本番環境のサイトを配置するとき、またはサイトで対応する必要のあるユーザーが増えたときには、ArcGIS Server コンピューターを追加できます。独自のエンタープライズ Web サーバー (ArcGIS Web Adaptor などを経由)、データベース、またはエンタープライズ ID プロバイダーを使用して、サイトを既存の IT インフラストラクチャに統合することもできます。さらに、可用性の高い構成を使用することで重要な事業活動をサポートするように ArcGIS Server を構成することもできます。

このヘルプ トピックでは、さまざまな容量および可用性の要件に合わせて ArcGIS Server サイトを設計する方法をいくつか説明します。それぞれの配置シナリオを説明する際に次の用語が使用されます。

  • サイト - ArcGIS Server サイトは、ArcGIS Server コンピューターや ArcGIS Web Adaptor などの複数のコンポーネントから構成されています。これらのコンポーネントは、処理能力と冗長性を向上させるために、オプションで複数台のコンピューターに分散することができます。詳細な説明については、「ArcGIS Server サイトの詳細」をご参照ください。
  • ArcGIS Server - GIS Web サービスに発行されたリクエストに対する処理を実行するサイトの主要コンポーネント。ArcGIS Server は、マップの生成、ツールの実行、画像の配信、および ArcGIS が提供するその他多くの処理を実行できます。
  • リバース プロキシ サーバー - ネットワーク インフラストラクチャ内でクライアントとサーバーの間に配置される、組織内のオプションのサード パーティ製コンポーネント。受信したリクエストは、プロキシによって処理されます。プロキシは、クライアントの代わりに、目的のサーバーまたはサーバー上に存在するサービスと情報をやりとりします。ほとんどの組織が、サイトが直接クライアントに公開されないようにするためにプロキシ サーバーを設定します。詳細については、「ArcGIS Server でのリバース プロキシ サーバーの使用」をご参照ください。
  • ArcGIS Web Adaptor - サイトへの Web エントリ ポイントを構成できるオプションのコンポーネント。Web サーバーに統合され、受信したリクエストを ArcGIS Server コンピューター間に分散します。詳細については、「ArcGIS Web Adaptor について」をご参照ください。
  • ネットワーク ロード バランサー (NLB) - さまざまなホスト間でネットワーク トラフィックの負荷を分散する分散アルゴリズムを使用するサード パーティ製コンポーネント。Web サービスのスケーラビリティと可用性を向上させるのに役立ちます。また、NLB は通常、コンピューターの障害を検出し、利用可能なコンピューターにトラフィックを自動的に再分散することで、高可用性を実現します。
  • サーバー ディレクトリ - サービスをサポートする特定の種類のファイルを含む一連の ArcGIS Server ディレクトリ。マップ キャッシュ、検索インデックス、ジオプロセシング ジョブの結果などのファイルがあります。詳細については、「サーバー ディレクトリについて」をご参照ください。
  • 構成ストア - サイトに参加している ArcGIS Server のリストなど、サイトに関する構成情報が格納されているファイル ディレクトリ。サイトが機能するには、構成ストアが利用可能でなければなりません。詳細については、「構成ストアについて」をご参照ください。
  • データ - フィーチャクラス、ツール、画像、ロケーターなどの Web サービスをサポートするデータ。詳細については、「データを ArcGIS Server でアクセス可能にする」をご参照ください。

次のトピックで概説するシナリオは、ArcGIS Server サイトを構築するときに参考にすべき配置のガイドラインです。いずれかのシナリオに示されているとおりにサイトを構成することができますが、これらの構成は柔軟であり、特別なニーズやハードウェア リソースに合わせて調整することができます。

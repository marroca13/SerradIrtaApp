ArcGIS でのポータルの役割

Portal for ArcGIS は、ArcGIS システム内の情報の整理と共有に中心的な役割を果たします。ポータルは、ユーザーのマップとアプリケーションに対して、わかりやすく検索可能なリポジトリを提供します。また、新しいマップとアプリケーションの作成と共有にも役立ちます。

これらのマップとアプリケーションの豊富な情報の一部は、組織内のデータベースから生成されます。このデータベースの GIS コンテンツは、ArcGIS Server によってホストされる Web サービスを使用して、他のデバイスと共有されます。マップおよびアプリケーションの作成者は、ポータルを使用することで、これらの Web サービスを検索および使用できます。ポータルは、GIS ソフトウェアのトレーニングを受けなくとも、GIS コンテンツに簡単にアクセスできる方法を提供します。

一部の組織は、独自データを使用した完全に自社開発のアプリケーションを作成するために使用できる Web サービスのパッケージを公開しています。しかし、ArcGIS Online から詳細なベースマップ、地名検索、住所検索、およびその他の GIS Web サービスを独自のサービスに追加するのが大半のケースです。このシナリオでは、独自の ArcGIS Server サービスと、Esri クラウドでホストされつねに更新されているサービスを組み合わせます。

マップとアプリケーションがインターネットにアクセスできない場合は、Data Appliance for ArcGIS を使用して、通常 ArcGIS Online で使用するのと同じベースマップ、住所検索、およびその他の Web サービスを提供できます。このタイプの配置は、組織のファイアウォールの内側で完全にホストできます。

内部ユーザーと外部ユーザーをサポートする場合、ファイアウォールの内側にあるポータルと、Web で公開するポータルの 2 つのポータルを配置する方法があります。これは、データの場所と更新を詳細に制御したい場合や、データをオフサイトでホストできない場合に適しています。また、外部用のアプリケーションとサービスに対して、ArcGIS Online の組織のサブスクリプションを使用する方法もあります。

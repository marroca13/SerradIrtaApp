Acerca de ArcGIS Web Adaptor

ArcGIS Web Adaptor (IIS) permite la integración de ArcGIS Server con su servidor web existente. Consulte la página de requisitos del sistema para obtener más información sobre los sistemas operativos admitidos para ArcGIS Web Adaptor (IIS).

Web Adaptor es una aplicación que se ejecuta en su sitio web existente y reenvía solicitudes a sus equipos de ArcGIS Server. Sondea el sitio en un intervalo regular para saber qué equipos se han agregado o eliminado. Entonces reenvía el tráfico únicamente para los equipos participantes actuales. Cuando se prepare para exponer el sitio de ArcGIS Server a una audiencia externa, debe instalar Web Adaptor o implementar tecnología de seguridad y envío de solicitudes comparable.

Beneficios Web Adaptor

Web Adaptor proporciona las siguientes ventajas:

  • Le permite integrar ArcGIS Server con el servidor web existente de la organización. Al incluir un servidor web en su sitio, obtiene la capacidad de alojar las aplicaciones web que utilizan los servicios SIG.

  • Le permite ofrecer un único extremo que distribuya solicitudes entrantes a ArcGIS Server de su sitio.

  • Le permite exponer su ArcGIS Server a través del sitio web y puerto estándar de su organización. Utilice Web Adaptor si no desea que los usuarios vean el puerto predeterminado 6080 o el nombre predeterminado del sitio arcgis.

  • Le permite bloquear ArcGIS Server Administrator Directory y ArcGIS Server Manager desde la vista de los usuarios externos.

  • Permite impedir que los usuarios de ArcGIS Desktop puedan establecer conexiones de administración o publicación con ArcGIS Server.

  • Le permite usar el almacén de identidades y las políticas de seguridad de su organización en el nivel web. Por ejemplo, si está utilizando IIS, puede usar la autenticación de Windows integrada para limitar quién entra en el portal.También puede usar una infraestructura de clave pública (PKI) si el servidor Web tiene soporte integrado o extensible. Esto le permite proporcionar un inicio de sesión único u otra experiencia de autenticación personalizada al iniciar sesión para usar servicios, aplicaciones web y ArcGIS Server.

Escenarios de implementación de Web Adaptor

Web Adaptor se puede utilizar en diversas configuraciones del sitio de ArcGIS Server. Por ejemplo, en un sitio con un único equipo de ArcGIS Server, puede instalar Web Adaptor en el mismo equipo que ArcGIS Server, o transferirlo a un servidor web dedicado. En una implementación de varios equipos, puede tener un punto de entrada en su sitio si instala Web Adaptor en un servidor web único, o puede establecer redundancia en el nivel de servidor web si instala Web Adaptor en varios servidores web.

La versión de Web Adaptor siempre debe coincidir con la del sitio registrado de ArcGIS Server. Si ha configurado varios adaptadores web en un único servidor web, puede utilizar números de versión diferentes para todos los adaptadores web.

Legado:

En la versión 10.2 y en versiones anteriores, todos los Web Adaptor de un mismo servidor web debían utilizar el mismo número de versión; al ejecutar el programa de instalación se actualizaban todas las versiones existentes de Web Adaptor a la versión más reciente.

Web Adaptor es independiente de la plataforma de ArcGIS Server; por lo tanto, el Web Adaptor que implemente no tendrá que coincidir con la plataforma del sistema operativo de su sitio de ArcGIS Server. Por ejemplo, si tiene un sitio compuesto de equipos Linux, puede implementar ArcGIS Web Adaptor (IIS) para que trabaje con ArcGIS Server.

Para obtener una descripción detallada de las distintas maneras en las que puede integrar Web Adaptor en la arquitectura de su sitio existente, consulte Escenarios de implementación.

Acceder a los servicios con Web Adaptor instalado

Después de instalar y configurar Web Adaptor, las direcciones URL que puede utilizar para acceder a sus servicios cambiarán. A continuación se muestran algunos ejemplos de direcciones URL que cambiarían después de instalar Web Adaptor para utilizar el puerto 80:

Directorio de servicios (servicios web REST)

  • Sin Web Adaptor: http://gisserver.domain.com:6080/arcgis/rest/services.

  • Con Web Adaptor: http://webadaptorhost.domain.com/webadaptorname/rest/services. Por ejemplo, si el equipo en el que está alojado Web Adaptor tiene el nombre wa con el dominio myorg.net y Web Adaptor se llama arcgis, accederá al Directorio de servicios usando la dirección URL http://wa.myorg.net/arcgis/rest/services.

Servicios web de SOAP

  • Sin Web Adaptor: http://gisserver.domain.com:6080/arcgis/services.

  • Con Web Adaptor: http://webadaptorhost.domain.com/webadaptorname/services. Por ejemplo, si el equipo en el que está alojado Web Adaptor tiene el nombre wa con el dominio myorg.net y Web Adaptor se llama arcgis, accederá a los servicios web de SOAP usando la dirección URL http://wa.myorg.net/arcgis/services.

La experiencia de instalación de Web Adaptor

Web Adaptor tiene su propia guía de configuración e instalación que es independiente de la instalación de ArcGIS Server. Debe instalar Web Adaptor en un equipo que ejecuta un servidor web. Puede tratarse de un equipo en el que ya se esté ejecutando un sitio de ArcGIS Server o de un equipo diferente.

Para obtener más información sobre la configuración, consulte Vista general de la implementación de Web Adaptor con ArcGIS Server.

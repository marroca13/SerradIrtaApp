Situaciones de implementación

ArcGIS Server está diseñado para ser escalable; puede alojar implementaciones grandes y pequeñas. Cuando empieza a construir su sitio, es posible que desee iniciar con algo pequeño e instalar todos los componentes en un solo equipo. A medida que implemente el sitio de producción o necesite manejar más usuarios, puede agregar más equipos de ArcGIS Server. También puede integrar su sitio en la infraestructura de TI existente utilizando su propio servidor Web (por ejemplo, mediante ArcGIS Web Adaptor), base de datos o proveedores de identidad corporativa. ArcGIS Server también se puede configurar de modo que sea compatible con los procesos empresariales esenciales utilizando configuraciones de alta disponibilidad.

En este libro de ayuda se describen diferentes formas de arquitectura que puede utilizar en su sitio de ArcGIS Server para cumplir con diversos requisitos de capacidad y disponibilidad. Los siguientes términos se utilizan al explicar cada situación de implementación:

  • Sitio: un sitio de ArcGIS Server consta de varios componentes, como un equipo de ArcGIS Server y ArcGIS Web Adaptor. Estos componentes se pueden distribuir de forma opcional en varios equipos para aumentar la capacidad de procesamiento y la redundancia. Para obtener una descripción más detallada, consulte Dentro de un sitio de ArcGIS Server.
  • ArcGIS Server: el principal componente del sitio que realiza el trabajo para satisfacer las solicitudes emitidas a los servicios Web de SIG. ArcGIS Server puede generar mapas, ejecutar herramientas, guardar imágenes y realizar muchas otras operaciones de ArcGIS.
  • Servidor proxy inverso: componente opcional de terceros en su organización que se coloca entre un cliente y un servidor en una infraestructura de red. El proxy maneja las solicitudes entrantes e interactúa en nombre del cliente con el servidor deseado o con un servicio que reside en el servidor. La mayoría de los organizadores configuran un servidor proxy de modo que el sitio no esté directamente expuesto a los clientes. Consulte Utilizar un servidor proxy inverso con ArcGIS Server para obtener más información.
  • ArcGIS Web Adaptor: componente opcional que le permite configurar un punto de entrada Web a su sitio. Se integra con su servidor Web y distribuye las solicitudes entrantes entre los equipos de ArcGIS Server. Para obtener más información, consulte Acerca de ArcGIS Web Adaptor.
  • Balanceador de carga de red (NLB): componente de terceros que utiliza un algoritmo de distribución para equilibrar la carga del tráfico de red entre varios hosts, lo que contribuye a mejorar la escalabilidad y disponibilidad de los servicios web. Además, normalmente ofrece una alta disponibilidad al detectar fallos de equipo y redistribuir automáticamente el tráfico a los equipos disponibles.
  • Directorios de servidor: conjunto de directorios de ArcGIS Server que contienen determinados tipos de archivos que admiten sus servicios. Estos archivos incluyen cachés de mapa, índices de búsqueda y resultados de trabajos de geoprocesamiento. Para obtener más información, consulte Acerca de los directorios del servidor.
  • Almacén de configuración: directorio de archivos que contiene información de configuración sobre el sitio, como la lista de ArcGIS Server que participan en él. El almacenamiento de configuración debe estar disponible para que el sitio funcione. Para obtener más información, consulte Acerca del almacenamiento de configuración.
  • Datos: datos que admiten los servicios web, como clases de entidad, herramientas, imágenes y localizadores. Para obtener más información, consulte Hacer que sus datos sean accesibles para ArcGIS Server.

Las situaciones que se describen en los siguientes temas se presentan como guías de implementación que debe tener en cuenta para crear el sitio de ArcGIS Server. Aunque puede configurar su sitio exactamente como se presenta en uno de los escenarios, estas configuraciones son flexibles y se puede ajustar para satisfacer necesidades y recursos de hardware especializados.

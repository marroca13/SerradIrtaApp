Acerca de ArcGIS Web Adaptor

ArcGIS Web Adaptor (IIS) es un componente de Portal for ArcGIS que permite la integración de su portal con su servidor web existente y los mecanismos de seguridad de su organización. ArcGIS Web Adaptor es necesario para utilizar la autenticación de Windows integrada con su portal.

Web Adaptor es una aplicación que se ejecuta en su sitio web existente y reenvía solicitudes al equipo donde está alojado Portal for ArcGIS. La instalación IIS es compatible con IIS en Windows 7, 8, 8.1, 10, Server 2008, 2008 R2, Server 2012 y 2012 R2. Esto corresponde a las versiones de IIS 7.0, 7.5, 8.0, 8.5 y 10.

Beneficios Web Adaptor

Web Adaptor proporciona las siguientes ventajas:

  • Le permite integrar Portal for ArcGIS con el servidor web existente de la organización. Al incluir un servidor web en su sitio, obtiene la capacidad de alojar las aplicaciones web que utilizan los servicios SIG.

  • Le permite usar el almacén de identidades y las políticas de seguridad de su organización en el nivel web. Por ejemplo, si está utilizando IIS, puede usar la autenticación de Windows integrada para limitar quién entra en el portal.También puede usar una infraestructura de clave pública (PKI) o cualquier otro almacén de identidades para los que el servidor Web tenga soporte integrado o extensible. Esto le permite proporcionar un inicio de sesión único u otra experiencia de autenticación personalizada al iniciar sesión para usar servicios, aplicaciones web y Portal for ArcGIS.

  • Le permite exponer Portal for ArcGIS mediante un nombre de sitio diferente de arcgis predeterminado.

  • Le permite exponer Portal for ArcGIS a través del puerto 80 o 443.

    Nota:

    Solo puede usar Web Adaptor con el puerto 80 o el puerto 443. No se admite el uso de puertos diferentes.

Escenarios de implementación de Web Adaptor

La versión de Web Adaptor siempre debe coincidir con la de Portal for ArcGIS. Si ha configurado varios adaptadores web en un único servidor web, puede utilizar números de versión diferentes para todos los adaptadores web.

Legado:

En la versión 10.2 y en versiones anteriores, todos los Web Adaptor de un mismo servidor web debían utilizar el mismo número de versión; al ejecutar el programa de instalación se actualizaban todas las versiones existentes de Web Adaptor a la versión más reciente.

Web Adaptor es independiente de la plataforma de Portal for ArcGIS; por lo tanto, el Web Adaptor que implemente no tendrá que coincidir con la plataforma de sistema operativo de su portal. Por ejemplo, si su portal se ejecuta en Linux, puede implementar ArcGIS Web Adaptor (IIS) para trabajar con Portal for ArcGIS.

Acceso a su portal con Web Adaptor instalado

Después de instalar y configurar Web Adaptor, la dirección URL que utilice para acceder a su portal tendrá el formato https://webadaptorhost.domain.com/webadaptorname/home. Por ejemplo, si el equipo en el que está alojado Web Adaptor tiene el nombre wa con el dominio myorg.net y Web Adaptor se llama arcgis, accederá al portal usando la dirección URL https://wa.myorg.net/arcgis/home.

Si ha configurado el portal para que utilice HTTPS para todas las comunicaciones, debería actualizar el sitio web del portal instalado y ayudar a que las direcciones URL de los accesos directos usen https en lugar de http; de lo contrario, se producirán fallos en su navegador cuando intente acceder a las direcciones URL de accesos directos originales.

La experiencia de instalación de Web Adaptor

Web Adaptor tiene su propia guía de configuración e instalación que es independiente de la instalación de Portal for ArcGIS. Debe instalar Web Adaptor en un equipo que ejecuta un servidor web. Puede tratarse de un equipo en el que ya se esté ejecutando Portal for ArcGIS o de un equipo diferente.

Para obtener más información sobre la configuración, consulte Vista general de la implementación de Web Adaptor con Portal for ArcGIS.

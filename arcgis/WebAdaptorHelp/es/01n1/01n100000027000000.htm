Acerca de los archivos de política de dominio cruzado

Los archivos de política de acceso cruzado permiten acceder a servicios Web fuera del dominio de la aplicación. Por defecto, a las aplicaciones Web de Adobe Flash y Microsoft Silverlight no se les permite acceder a los servicios Web que residen fuera del dominio donde se aloja la aplicación Web.

Cuando la opción Archivos de política de dominio cruzado está habilitada, se instalarán dos archivos de política de dominio cruzado en la carpeta raíz del servidor web de IIS: crossdomain.xml para aplicaciones de Adobe Flash y clientaccesspolicy.xml para aplicaciones de Microsoft Silverlight.

Puede limitar el acceso a determinados recursos del servidor web (por ejemplo, carpetas, archivos, tipos de solicitudes y protocolos) creando o modificando un archivo de política de dominio cruzado. Para obtener información sobre cómo hacerlo, consulte:

  • Para obtener información sobre cómo crear o modificar un archivo crossdomain.xml, consulte la especificación del archivo de política de dominio cruzado de Adobe.
  • Para crear o modificar un archivo clientaccesspolicy.xml, consulte Hacer que un servicio esté disponible a través de los límites de dominio y restricciones de acceso de seguridad de red en Microsoft Silverlight.

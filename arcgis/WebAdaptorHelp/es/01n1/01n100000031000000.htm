Deshabilitar la búsqueda de grupos de Active Directory de Windows en ArcGIS Web Adaptor (IIS)

Cuando realiza una autenticación de nivel web con ArcGIS Web Adaptor (IIS), Web Adaptor busca los grupos de Active Directory de Windows correspondientes al usuario conectado cada vez que se envía una solicitud al sitio de ArcGIS Server. En organizaciones con un número pequeño de grupos, esta búsqueda no afecta en modo alguno al rendimiento. No obstante, si la organización dispone de cientos o miles de grupos, puede que observe una disminución del rendimiento debido al tiempo que se necesita para realizar la búsqueda.

Si observa una disminución del rendimiento, puede deshabilitar la búsqueda de grupos de Active Directory en ArcGIS Web Adaptor (IIS). Esta función solo puede deshabilitarse si se cumplen los siguientes criterios:

  • Ha configurado la autenticación de nivel web (autenticación de Windows integrada o PKI) en ArcGIS Web Adaptor (IIS).
  • Está utilizando los grupos integrados de ArcGIS Server como almacén de grupos para el sitio. Si está utilizando grupos de Active Directory de Windows, la búsqueda debe realizarse para obtener grupos.

Para deshabilitar la búsqueda de grupos de Active Directory en ArcGIS Web Adaptor (IIS), proceda tal como se indica a continuación:

Pasos:
  1. En el equipo donde se aloja ArcGIS Web Adaptor (IIS), desplácese a la ubicación en la que se encuentra el archivo WebAdaptor.config y ábralo. De manera predeterminada, este archivo está ubicado en la carpeta inetpub de IIS; por ejemplo, C:\inetpub\wwwroot\<web adaptor name>\WebAdaptor.config.
  2. Localice la propiedad EnableGetRolesForUser y cambie el valor a falso, por ejemplo:
    <EnableGetRolesForUser>false</EnableGetRolesForUser>
  3. Guarde y cierre el archivo
  4. Reinicie IIS.
  5. Repita estos pasos con el resto de Web Adaptors que haya configurado con el sitio.

Puede habilitar las búsquedas en cualquier momento cambiando la propiedad EnableGetRolesForUser a true.

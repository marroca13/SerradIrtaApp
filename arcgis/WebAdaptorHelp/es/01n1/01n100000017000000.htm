Configurar ArcGIS Web Adaptor

Después de instalar ArcGIS Web Adaptor, debe configurarlo para trabajar con un sitio de ArcGIS Server. Esto se puede hacer desde una página web o mediante la utilidad línea de comandos instalada con Web Adaptor. Como medida de seguridad, solo se puede acceder a la página de configuración y a la utilidad de línea de comandos desde el equipo donde está alojado Web Adaptor.

Al abrir la página de configuración se muestra el estado actual de Web Adaptor. Esto indica si Web Adaptor se configuró previamente. Para ver el tipo de plataforma y el número de versión de Web Adaptor, haga clic en Acerca de en la página de configuración.

Para obtener instrucciones completas para configurar Web Adaptor, consulte una de las secciones siguientes.

Configurar Web Adaptor desde la página web de configuración

Para configurar Web Adaptor desde la página web de configuración instalada con el software, siga los pasos siguientes.

Pasos:
  1. Abra la página de configuración del Web Adaptor en un navegador web. La dirección URL tiene el formato siguiente: http://webadaptorhost.domain.com/webadaptorname/webadaptor.
  2. Seleccione ArcGIS Server y haga clic en Siguiente.
  3. Para Dirección URL de ArcGIS Server, escriba la dirección URL para uno de los equipos de ArcGIS Server en su sitio de ArcGIS Server. Esta dirección URL se utilizará para descubrir todos los equipos que participan en el sitio y registrarlos con Web Adaptor. Por ejemplo, la dirección URL debe tomar la forma http://gisserver.domain.com:6080.
  4. Especifique el nombre de usuario y la contraseña para una cuenta que tenga privilegios administrativos en el sitio de ArcGIS Server. Normalmente se utilizan el nombre de usuario y la contraseña de la cuenta del administrador principal del sitio definida al crear el sitio. Si la cuenta del administrador principal del sitio está desactivada, se deberá especificar una cuenta con acceso de administración al sitio.
  5. Elija si los usuarios pueden administrar el sitio a través de Web Adaptor. De forma predeterminada, la administración del sitio mediante Web Adaptor está desactivada. Ésta es la opción recomendada. A continuación encontrará unas consideraciones a tener en cuenta en cuanto se refiere a esta opción:
    • Cuando la administración está desactivada, los usuarios externos tienen bloqueado el acceso a ArcGIS Server Manager y ArcGIS Server Administrator Directory a través de Web Adaptor. Además, los usuarios de ArcGIS Desktop tienen bloqueado el acceso a establecer conexiones de administración o publicación con ArcGIS Server. Tenga en cuenta que siguen siendo posibles las conexiones de los usuarios desde ArcGIS Desktop a ArcGIS Server tanto si esta opción está activada como si está desactivada.
    • Cuando la opción de acceso administrativo a través de Web Adaptor está desactivada, es posible acceder al Administrador y al Directorio del administrador, así como conectarse al servidor en ArcGIS Desktop utilizando una dirección URL directa a uno de los equipos de ArcGIS Server de su sitio. Por ejemplo
      • Para acceder al Administrador, utilice el formato de URL http://gisserver.domain.com:6080/arcgis/manager.
      • Para acceder al Directorio del administrador, utilice el formato de URL http://gisserver.domain.com:6080/arcgis/admin.
      • Para crear una conexión al servidor nueva en ArcGIS Desktop, utilice el formato de URL http://gisserver.domain.com:6080/arcgis.
    • Si ArcGIS Server está configurado con autenticación de nivel web, debe habilitar la administración mediante Web Adaptor. Esto permite a los usuarios del almacén de identidades corporativas con privilegios de publicador y administrador publicar servicios desde ArcGIS Desktop. Cuando los usuarios con estos roles se conectan al servidor en ArcGIS Desktop, deben especificar la dirección URL de Web Adaptor.
  6. Haga clic en Configurar para aplicar la configuración del adaptador Web.

Cuando la configuración es correcta, verá una lista de equipos de ArcGIS Server reconocidos por Web Adaptor en la parte inferior de la página de configuración (resaltada en verde). Puede volver a esta página en cualquier momento para ver la lista de los equipos reconocido y ajustar la configuración del Web Adaptor.

Configurar Web Adaptor desde la línea de comandos

Puede configurar Web Adaptor mediante la utilidad de la línea de comandos (ConfigureWebAdaptor.exe) de C:\Program Files (x86)\Common Files\ArcGIS\WebAdaptor\IIS\Tools.

Los parámetros disponibles son los siguientes:

ConfigureWebAdaptor.exe/m <Mode>/w <WebAdaptorURL>/g <URL>/u <Username>/p <Password>/a <AdminAccessEnabled>

  • <Mode>: modo de producto en el que se configurará Web Adaptor. De forma predeterminada, este parámetro se establece en servidor, lo que significa que Web Adaptor se configurará para usarse con ArcGIS Server. Si está configurando Web Adaptor para usarse con Portal for ArcGIS, especifique este parámetro como portal.

  • <WebAdaptorURL>: dirección URL de Web Adaptor que se desea configurar desde la línea de comandos. Es la misma URL que se utilizaría para acceder a la página de configuración de Web Adaptor desde un navegador web. Si Web Adaptor se ha instalado en un puerto distinto del puerto 80 predeterminado, deberá especificar el puerto en la URL.

  • <Dirección URL>: la dirección URL a uno de los equipos de ArcGIS Server de su sitio de ArcGIS Server. Esta dirección URL se utilizará para descubrir todos los equipos que participan en el sitio y registrarlos con Web Adaptor.

  • <Nombre de usuario>: el nombre de usuario de una cuenta con privilegios de administración sobre el sitio de ArcGIS Server. Normalmente se utiliza la cuenta del administrador principal del sitio definida al crear el sitio.

  • <Contraseña>: la contraseña de una cuenta con privilegios de administración sobre el sitio de ArcGIS Server. Normalmente se utiliza la cuenta del administrador principal del sitio definida al crear el sitio.

  • <AdminAccessEnabled>: indica si ArcGIS Server Manager y ArcGIS Server Administrator Directory estarán disponibles a través de Web Adaptor. Como medida de seguridad, de forma predeterminada este parámetro está ajustado a false. Para habilitar el acceso de administración, ajuste este parámetro a true.

Ejemplo: ConfigureWebAdaptor.exe /m server /w http://webadaptorhost.domain.com/webadaptorname/webadaptor /g http://gisserver.domain.com:6080 /u siteadmin /p secret /a false

Legado:

En la versión 10.2 y en versiones anteriores era necesario incorporar /server al final del parámetro de la dirección URL de Web Adaptor, por ejemplo, /w http://webadaptorhost.domain.com/webadaptorname/webadaptor/server. Esto ya no es necesario a partir de la versión 10.2.1.

Después de configurar Web Adaptor

Una vez que Web Adaptor está configurado para el uso, se deben tener en cuenta los elementos importantes siguientes:

Agregar o quitar equipos de ArcGIS Server.

Al agregar o quitar equipos de ArcGIS Server del sitio, Web Adaptor tarda un minuto en reconocer los cambios realizados. Si desea que Web Adaptor reconozca inmediatamente las máquinas agregadas o quitadas, puede reconfigurarlo siguiendo las instrucciones anteriores.

Instalar múltiples Web Adaptor

Es posible proporcionar distintos puntos de entrada web en su sitio de ArcGIS Server instalando múltiples Web Adaptor. Esto le puede interesar si desea aumentar la disponibilidad del sitio para los clientes, proporcionar soporte para aplicaciones anteriores o permitir el acceso de publicación y administración a usuarios específicos dentro de un entorno de alta seguridad. Para obtener más información, consulte las Situaciones de implementación.

Es posible instalar múltiples Web Adaptor en un mismo equipo simultáneamente o repartirlos en equipos separados. Si desea instalar múltiples Web Adaptor en un mismo sitio web (puerto), deben utilizar nombres distintos. Por ejemplo, en un mismo sitio web no puede haber dos Web Adaptor llamados arcgis. Si desea disponer de múltiples Web Adaptor con un mismo nombre, deberá instalarlos en sitios web (puertos) separados.

Habilitar HTTPS en ArcGIS Server

De forma predeterminada, ArcGIS Server utiliza los protocolos HTTP y HTTPS para la comunicación. Si actualiza el protocolo de comunicación de ArcGIS Server solo a HTTP o solo a HTTPS, Web Adaptor tarda un minuto en detectar los cambios del sitio.

Legado:

En la versión 10.2.1 y anteriores, era necesario volver a configurar ArcGIS Web Adaptor después de actualizar el protocolo de comunicación de ArcGIS Server. Esto ya no es necesario a partir de la versión 10.2.2.

Para obtener más información sobre el protocolo de comunicación de ArcGIS Server, consulte Asegurar la comunicación de ArcGIS Server.

Para obtener instrucciones sobre cómo instalar varios Web Adaptor, consulte Instalar múltiples Web Adaptor.

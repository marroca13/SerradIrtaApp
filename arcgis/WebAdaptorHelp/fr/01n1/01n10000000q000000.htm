Installation d'ArcGIS Web Adaptor en mode silencieux

Il est possible d'installer ArcGIS Web Adaptor en exécutant le programme d'installation à l'aide des paramètres de ligne de commande de Windows Installer, comme cela est décrit ci-dessous. Vous pouvez également procéder à une installation à partir de l'interface utilisateur à l'aide de Setup.exe.

Conditions à remplir pour l'installation

La version d'ArcGIS Web Adaptor doit toujours correspondre à celle de ArcGIS Server que vous utilisez.

Vous pouvez installer la version 10.2.1 ou une version ultérieure d'ArcGIS Web Adaptor sur une machine sur laquelle est installée une version antérieure d'ArcGIS Web Adaptor. Vous pouvez, par exemple, installer la version 10.1 SP1 et une version 10.6 d'ArcGIS Web Adaptor sur le même serveur Web.

Chaque adaptateur Web doit avoir un nom unique. Deux adaptateurs Web du même nom ne peuvent pas coexister sur le même serveur Web. En présence d'un conflit d'attribution de nom pendant l'installation, un message d'avertissement apparaît. Vous devez désinstaller la version antérieure du même nom pour résoudre le conflit. Vous pouvez également modifier le nom d'un adaptateur Web pour poursuivre l'installation.

Services Internet Microsoft (IIS) est requis. Uniquement dans le cas d'une installation en mode silencieux, si IIS est introuvable, le programme d'installation d'ArcGIS Web Adaptor n'aboutit pas. IIS et tous les composants nécessaires doivent être installés avant l'installation d'ArcGIS Web Adaptor en mode silencieux.

Microsoft .NET Framework 4.5 ou une version supérieure est requise. Le programme d'installation s'arrêtera si Microsoft .NET Framework 4.5 ou une version supérieure est introuvable.

Héritage :

Dans les versions antérieures, ArcGIS Web Adaptor nécessitait l'installation préalable de Microsoft .NET Framework 3.5 Service Pack 1 (SP1), 4.0 ou 4.5.

Une fois ArcGIS Web Adaptor installé, il doit être configuré. Pour plus d'informations, reportez-vous à la rubrique Configuration d'ArcGIS Web Adaptor.

Mettre à niveau une version antérieure d'ArcGIS Web Adaptor

Pour mettre à niveau une version précédente d’ArcGIS Web Adaptor vers la version 10.6, vous devez désinstaller la version précédente et installer la version 10.6. Pendant l’installation de la version 10.6, vous devez indiquer le nom de l'adaptateur Web précédent. Ainsi, l'URL de l'adaptateur Web sera identique à celle de la version précédente.

En présence d'un conflit d'attribution de nom pendant l'installation de la version 10.6, un message d'avertissement apparaît. Vous devez désinstaller la version antérieure du même nom pour procéder à la mise à niveau.

Après avoir installé la version 10.6, vous devez configurer ArcGIS Web Adaptor avec son site ArcGIS Server correspondant.

Héritage :

Dans les versions 10.2 et antérieures, il suffisait d'exécuter le fichier d'installation pour mettre à niveau toutes les versions existantes des adaptateurs Web.

Paramètres de ligne de commande pour indiquer le site web et le nom d'ArcGIS Web Adaptor

Les lignes de commande suivantes sont utilisées pour configurer IIS. Si ces paramètres de ligne de commande ne sont pas appelés lors d'une installation en mode silencieux, l'application web ArcGIS Web Adaptor est créée sous le site web par défaut avec le nom arcgis, par défaut.

  • WEBSITE_ID=<valeur numérique du site Web>

    Le paramètre WEBSITE_ID permet de spécifier le site Web où l'application Web ArcGIS Web Adaptor (IIS) sera créée. La valeur de WEBSITE_ID est un nombre spécifique attribué par IIS à chaque site Web sur votre serveur. Les identifiants pour les sites Web sur votre serveur se trouvent dans le gestionnaire de services Internet (IIS). Le site Web par défaut possède un identifiant de site Web égal à 1. Par défaut, le paramètre WEBSITE_ID est défini sur le site Web par défaut (même si plusieurs sites Web existent). Le paramètre WEBSITE_ID respecte la casse.

  • PORT=<numéro de port>

    En outre, si votre site Web dispose de plusieurs ports, utilisez le paramètre PORT pour effectuer l'installation sur le port spécifié WEBSITE_ID.

    Par exemple, le chemin est le suivant : <path to ArcGIS Web Adaptor (IIS) setup download>\setup.exe /qb VDIRNAME=arcgis_external WEBSITE_ID=4059640 PORT=82.

  • VDIRNAME=<nom d'ArcGIS Web Adaptor>

    Ce paramètre de ligne de commande est facultatif. Par défaut, la propriété VDIRNAME est définie sur arcgis. Si vous n'activez pas le paramètre VDIRNAME dans votre ligne de commande, l'Adaptateur Web est créé en tant qu'arcgis. Le paramètre VDIRNAME respecte la casse. Le nom de l'adaptateur Web ne doit pas comporter d'espace.

    Par exemple, le chemin est le suivant : <path to ArcGIS Web Adaptor (IIS) setup download>\setup.exe /qb VDIRNAME=arcgis_external WEBSITE_ID=4059640.

  • CONFIGUREIIS=TRUE

    Ce paramètre facultatif permet au programme d’installation d'effectuer l’installation et la configuration en mode silencieux de tous les composants Microsoft IIS manquants requis par ArcGIS Web Adaptor.

A propos d'ArcGIS Web Adaptor

ArcGIS Web Adaptor (IIS) est un composant de Portal for ArcGIS, qui vous permet d'intégrer votre portail dans votre serveur Web existant et dans les mécanismes de sécurité de votre organisation. ArcGIS Web Adaptor est requis pour utiliser l'authentification Windows intégrée avec votre portail..

ArcGIS Web Adaptor est une application qui s'exécute sur votre site Web et transmet des requêtes à la machine hébergeant Portal for ArcGIS. L'installation IIS est compatible avec IIS sous Windows 7, 8, 8.1, 10, Server 2008, 2008 R2, Server 2012 et 2012 R2. Cela correspond aux versions d'IIS 7.0, 7.5, 8.0, 8.5 et 10.

Avantages de l'adaptateur Web

L'adaptateur Web présente les avantages suivants :

  • Il permet d'intégrer Portal for ArcGIS au serveur Web de votre organisation. L'intégration d'un serveur Web à votre site vous permet d'héberger des applications Web qui utilisent vos services SIG.

  • Il vous permet d'utiliser le magasin d'identifiants et les stratégies de sécurité de votre organisation au niveau du Web. Par exemple, si vous utilisez IIS, vous pouvez faire appel à l'authentification Windows intégrée pour limiter les personnes autorisées à accéder au portail.Vous pouvez également utiliser une infrastructure à clé publique (PKI) ou un autre magasin d'identifiants pour lequel le serveur Web dispose d'une prise en charge intégrée ou extensible. Cela vous permet de fournir une authentification unique ou une autre authentification personnalisée en cas de connexion pour utiliser des services, des applications Web et Portal for ArcGIS.

  • Il vous permet de mettre Portal for ArcGIS à disposition à l'aide d'un nom de site autre que le nom arcgis par défaut.

  • Il vous permet de mettre Portal for ArcGIS à disposition par l'intermédiaire du port 80 ou 443.

    Remarque :

    Vous pouvez uniquement utiliser l'adaptateur Web avec le port 80 ou 443. L'utilisation d'autres ports n'est pas prise en charge.

Scénarios de déploiement de l'adaptateur Web

La version de l'adaptateur Web doit toujours correspondre à celle de Portal for ArcGIS. Si vous avez configuré plusieurs adaptateurs Web sur un seul serveur Web, vous pouvez utiliser des numéros de version différents pour tous les adaptateurs Web.

Héritage :

Pour la version 10.2 et les versions antérieures, tous les adaptateurs Web installés sur un serveur Web unique devaient porter le même numéro de version. Lorsque vous exécutiez le programme d'installation, tous les adaptateurs Web sont mis à niveau.

L'adaptateur Web ne dépendant pas de la plateforme de Portal for ArcGIS, celui que vous déployez ne doit pas nécessairement correspondre à la plateforme du système d'exploitation de votre portail. Par exemple, si votre portail s'exécute sous Linux, vous pouvez déployer ArcGIS Web Adaptor (IIS) pour l'utiliser avec Portal for ArcGIS.

Accès au portail après l'installation de l'adaptateur Web

Après l'installation et la configuration de l'adaptateur Web, l'URL que vous utilisez pour accéder à votre portail se présente comme suit : https://webadaptorhost.domain.com/webadaptorname/home. Par exemple, si la machine qui héberge votre adaptateur Web se nomme wa avec le domaine myorg.net et que votre adaptateur Web s'appelle arcgis, vous accéderez au portail via l'URL https://wa.myorg.net/arcgis/home.

Si vous avez configuré votre portail pour qu'il utilise le protocole HTTPS dans toutes les communications, vous devez mettre à jour le site Web du portail installé et les URL des raccourcis d'aide de manière à utiliser https au lieu de http. Sinon, des échecs se produiront lorsque votre navigateur essaiera d'accéder aux URL des raccourcis d'origine.

Procédure d'installation de l'adaptateur Web

L'adaptateur Web a son propre guide d'installation et de configuration, qui est distinct de l'installation de Portal for ArcGIS. Vous devez installer l'adaptateur Web sur une machine exécutant un serveur Web. Il peut s'agir d'une machine exécutant déjà Portal for ArcGIS ou encore d'une machine distincte.

Pour plus d'informations sur la configuration, reportez-vous à la rubrique Vue d'ensemble de l'implémentation de Web Adaptor avec Portal for ArcGIS.

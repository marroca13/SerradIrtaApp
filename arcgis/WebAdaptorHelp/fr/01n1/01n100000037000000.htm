Configurer plusieurs instances d'ArcGIS Web Adaptor

Plusieurs adaptateurs ArcGIS Web Adaptor sont nécessaires pour configurer un portail haute disponibilité utilisant l'authentification au niveau du Web. Pour configurer plusieurs adaptateurs Web avec Portal for ArcGIS, un seul point de terminaison doit être utilisé pour accéder au portail. Cette URL doit être définie dans la propriété WebContextURL . Il est possible de configurer plusieurs adaptateurs Web avec le portail en suivant la procédure ci-après.

Etapes :
  1. Configurez la première instance d'ArcGIS Web Adaptor avec votre portail.
  2. Ajoutez Portal for ArcGIS à votre serveur proxy inverse. Vérifiez que la propriété WebContextURL est définie comme suit :
    1. Ouvrez un navigateur Web et connectez-vous au répertoire Portal for ArcGIS en tant qu'administrateur de votre organisation. L'URL est au format https://portal.domain.com:7443/arcgis/portaladmin.
    2. Cliquez sur Système > Propriétés > Mettre à jour les propriétés. Ceci entraîne le redémarrage de Portal for ArcGIS. Vous devrez attendre que le redémarrage soit terminé pour continuer.
    3. Dans la boîte de dialogue Mettre à jour les propriétés du système, insérez la notation JSON suivante en remplaçant l'URL de l'alias DNS ou de votre serveur proxy inverse, telle qu'elle apparaît aux utilisateurs en dehors du pare-feu de votre organisation.
      {
   "WebContextURL": "https://reverseproxy.domain.com/myorg"
}
    4. Cliquez sur Mettre à jour les propriétés.
  3. Configurez des instances d'ArcGIS Web Adaptor supplémentaires.

Toutes les instances d'ArcGIS Web Adaptor configurées avec le portail sont répertoriées sur la page du récapitulatif de la configuration. Les utilisateurs accèdent au site du portail via le serveur proxy inverse ou l'URL de l'alias DNS.

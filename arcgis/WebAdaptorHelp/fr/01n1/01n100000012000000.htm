A propos d'ArcGIS Web Adaptor

ArcGIS Web Adaptor (IIS) permet l'intégration d'ArcGIS Server dans votre serveur Web existant. Reportez-vous à la page Configuration système requise pour de plus amples informations sur les systèmes d'exploitation pris en charge par ArcGIS Web Adaptor (IIS).

ArcGIS Web Adaptor est une application qui s'exécute sur votre site Web et transmet des requêtes à vos machines ArcGIS Server. Il interroge votre site à intervalle régulier pour savoir quelles machines ont été ajoutées ou supprimées. Il achemine ensuite le trafic uniquement aux machines actives. Lorsque vous préparez votre site ArcGIS Server en vue de le proposer à un public externe, nous vous recommandons d'installer l'adaptateur Web ou des technologies comparables de transmission des requêtes et de sécurisation.

Avantages de l'adaptateur Web

L'adaptateur Web présente les avantages suivants :

  • Il permet d'intégrer ArcGIS Server au serveur Web de votre organisation. L'intégration d'un serveur Web à votre site vous permet d'héberger des applications Web qui utilisent vos services SIG.

  • Il permet de fournir une extrémité unique qui distribue les requêtes entrantes aux serveurs ArcGIS Server de votre site.

  • Il permet de mettre votre serveur ArcGIS Server à disposition via le port et le site Web standard de votre organisation. Utilisez l'adaptateur Web si vous ne souhaitez pas que les utilisateurs voient le port par défaut 6080 ni le nom de site arcgis par défaut.

  • Il permet également d'empêcher les utilisateurs externes d'afficher le répertoire administrateur d'ArcGIS Server et ArcGIS Server Manager.

  • Il permet d'empêcher les utilisateurs d'ArcGIS Desktop d'établir des connexions administrateur ou éditeur à ArcGIS Server.

  • Il vous permet d'utiliser le magasin d'identifiants et les stratégies de sécurité de votre organisation au niveau du Web. Par exemple, si vous utilisez IIS, vous pouvez faire appel à l'authentification Windows intégrée pour limiter les personnes autorisées à accéder au portail.Vous pouvez également utiliser une infrastructure à clé publique (PKI) si le serveur Web dispose d'une prise en charge intégrée ou extensible. Cela vous permet de fournir une authentification unique ou une autre authentification personnalisée en cas de connexion pour utiliser des services, des applications Web et ArcGIS Server.

Scénarios de déploiement de l'adaptateur Web

L'adaptateur Web peut être utilisé dans plusieurs configurations de site ArcGIS Server. Par exemple, dans un site doté d'une seule machine ArcGIS Server, vous pouvez installer l'adaptateur Web sur la même machine que la machine ArcGIS Server ou le déplacer sur un serveur Web dédié. Dans un déploiement comprenant plusieurs machines, vous pouvez définir un point d'entrée dans votre site en installant l'adaptateur Web sur un seul serveur Web existant. Vous pouvez également établir une redondance au niveau du serveur Web en installant l'adaptateur Web sur plusieurs serveurs Web.

La version de l'adaptateur Web doit toujours correspondre à celle de son site ArcGIS Server inscrit. Si vous avez configuré plusieurs adaptateurs Web sur un seul serveur Web, vous pouvez utiliser des numéros de version différents pour tous les adaptateurs Web.

Héritage :

avant la version 10.2, tous les adaptateurs Web installés sur un serveur Web unique devaient porter le même numéro de version. Lorsque vous exécutiez le programme d'installation, tous les adaptateurs Web étaient mis à niveau.

L'adaptateur Web ne dépendant pas de la plateforme de ArcGIS Server, celui que vous déployez ne doit pas nécessairement correspondre à la plateforme du système d'exploitation de votre site ArcGIS Server. Par exemple, si votre site se compose de machines Linux, vous pouvez déployer ArcGIS Web Adaptor (IIS) pour l'utiliser avec ArcGIS Server.

Pour obtenir une description détaillée des différentes méthodes d'intégration de l'adaptateur Web dans l'architecture de votre site, reportez-vous à la rubrique Scénarios de déploiement.

Accès aux services après l'installation de l'adaptateur Web

Après l'installation et la configuration de l'adaptateur Web, les URL que vous utilisez pour accéder à vos services changent. Vous trouverez ci-après des exemples d'URL qui changent après l'installation d'un adaptateur Web utilisant le port 80.

Répertoire des services (services Web REST)

  • Sans l'adaptateur Web : http://gisserver.domain.com:6080/arcgis/rest/services.

  • Avec l'adaptateur Web : http://webadaptorhost.domain.com/webadaptorname/rest/services. Par exemple, si la machine qui héberge votre adaptateur Web se nomme wa avec le domaine myorg.net et que votre adaptateur Web s'appelle arcgis, vous accéderez au répertoire des services via l'URL http://wa.myorg.net/arcgis/rest/services.

Services Web SOAP

  • Sans l'adaptateur Web : http://gisserver.domain.com:6080/arcgis/services.

  • Avec l'adaptateur Web : http://webadaptorhost.domain.com/webadaptorname/services. Par exemple, si la machine qui héberge votre adaptateur Web se nomme wa avec le domaine myorg.net et que votre adaptateur Web s'appelle arcgis, vous accéderez aux services Web SOAP via l'URL http://wa.myorg.net/arcgis/services.

Procédure d'installation de l'adaptateur Web

L'adaptateur Web a son propre guide d'installation et de configuration, qui est distinct de l'installation de ArcGIS Server. Vous devez installer l'adaptateur Web sur une machine exécutant un serveur Web. Il peut s'agir d'une machine exécutant déjà un site ArcGIS Server ou encore d'une machine distincte.

Pour plus d'informations sur la configuration, reportez-vous à la rubrique Vue d'ensemble de l'implémentation de Web Adaptor avec ArcGIS Server.

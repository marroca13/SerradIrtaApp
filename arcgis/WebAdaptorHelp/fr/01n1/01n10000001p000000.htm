Configurer ArcGIS Web Adaptor

Après avoir installé ArcGIS Web Adaptor, vous devez le configurer pour qu'il fonctionne avec votre ArcGIS Enterprise portail. Pour ce faire, utilisez une page Web de configuration ou un utilitaire de ligne de commande installé avec ArcGIS Web Adaptor. Par mesure de sécurité, la page de configuration est accessible uniquement à partir de la machine qui héberge ArcGIS Web Adaptor.

L'état actuel d'ArcGIS Web Adaptor s'affiche lorsque vous ouvrez la page de configuration. Il indique si ArcGIS Web Adaptor a déjà été configuré. Pour afficher le type de plateforme et le numéro de version d'ArcGIS Web Adaptor, cliquez sur A propos de dans la page de configuration.

Si vous avez déjà configuré un adaptateur Web avec ArcGIS Enterprise votre portail et souhaitez mettre votre portail à jour pour utiliser un autre adaptateur Web, vous devez d'abord désinscrire l’adaptateur Web en cours. Pour procéder à une désinscription vous devez passer par le répertoire du portail. La fonctionnalité de désinscription n'est pas disponible à partir de la page de configuration ou du site Web du portail. Pour en savoir plus, reportez-vous à la rubrique Désinscription d'ArcGIS Web Adaptor avec Portal for ArcGIS.

Pour obtenir des instructions complètes sur la configuration d'ArcGIS Web Adaptor, procédez comme suit.

Attention :

Avant de continuer, vérifiez que ArcGIS Web Adaptor est installé sur un site web s'exécutant sur le port 80 ou 443. ArcGIS Web Adaptor ne peut pas être configuré avec Portal for ArcGIS s'il est installé sur un site Web s'exécutant sur des ports autres que le 80 ou 443.

Configurer ArcGIS Web Adaptor à partir de la page Web de configuration

Pour configurer ArcGIS Web Adaptor à partir de la page Web de configuration installée avec le logiciel, suivez la procédure ci-après.

Etapes :
  1. Activez HTTPS sur le port 443 de votre serveur Web si tel n'est pas encore le cas. Pour en savoir plus, reportez-vous à la rubrique Activation de HTTPS sur votre serveur Web.
  2. Ouvrez la page de configuration dans un navigateur Web via une URL HTTPS, telle que https://webadaptorhost.domain.com/webadaptorname/webadaptor.

    Si un alias DNS doit être utilisé avec le portail, ArcGIS Web Adaptor doit être configuré via l'alias DNS, à l'aide d'une URL telle que https://<dnsalias.domain.com>/<webadaptorname>/webadaptor.

  3. Choisissez Portal for ArcGIS et cliquez sur Suivant.
  4. Pour URL du portail, saisissez l'URL de la machine qui héberge Portal for ArcGIS. Incluez le nom de domaine complet de la machine dans l'URL, par exemple https://portal.domain.com:7443.
  5. Indiquez le nom d'utilisateur et le mot de passe d'un compte qui dispose des privilèges d'administrateur sur Portal for ArcGIS. Vous utilisez généralement le nom d'utilisateur et le mot de passe du compte d'administrateur initial que vous avez défini lors de la première configuration de votre portail. Si vous rétrogradez ou supprimez le compte d'administrateur initial, vous devez spécifier un compte disposant d'un accès administrateur au site Web du portail.
  6. Cliquez sur Configurer.

ArcGIS Web Adaptor est maintenant configuré de manière à être utilisé avec la machine qui héberge Portal for ArcGIS. Vous accédez désormais à votre portail par l'intermédiaire de l'URL d'ArcGIS Web Adaptor, au lieu du port 7443. L'URL se présente comme suit : https://webadaptorhost.domain.com/webadaptorname/home.

Si vous avez configuré votre portail pour qu'il utilise le protocole HTTPS dans toutes les communications, vous devez mettre à jour le site Web du portail installé et les URL des raccourcis d'aide de manière à utiliser https au lieu de http. Sinon, des échecs se produiront lorsque votre navigateur essaiera d'accéder aux URL des raccourcis d'origine.

Si vous comptez configurer Portal for ArcGIS avec le serveur proxy inversé de votre organisation, vous devez fournir certaines informations sur votre portail concernant le serveur proxy. Pour des instructions complètes, reportez-vous à la rubrique Configuration d'un serveur proxy inversé avec votre portail.

Configuration d'ArcGIS Web Adaptor à partir de la ligne de commande

Vous pouvez configurer ArcGIS Web Adaptor (IIS) grâce à l'utilitaire de ligne de commande (ConfigureWebAdaptor.exe) dans C:\Program Files (x86)\Common Files\ArcGIS\WebAdaptor\IIS\Tools.

Les paramètres disponibles sont les suivants :

ConfigureWebAdaptor.exe /m <Mode> /w <WebAdaptorURL> /g <URL> /u <Username> /p <Password>

  • <Mode> : mode du produit dans lequel ArcGIS Web Adaptor sera configuré. Par défaut, ce paramètre est défini sur serveur, ce qui signifie qu'ArcGIS Web Adaptor sera configuré pour fonctionner avec ArcGIS Server. Comme vous configurez ArcGIS Web Adaptor de manière à l'utiliser avec Portal for ArcGIS, définissez ce paramètre sur portail.

  • <URLWebAdaptor> : URL d'ArcGIS Web Adaptor que vous souhaitez configurer à partir de la ligne de commande. Il s'agit de l'URL à utiliser pour accéder à la page de configuration dans un navigateur Web. Vous devez spécifier l'URL HTTPS d'ArcGIS Web Adaptor, par exemple https://webadaptorhost.domain.com/webadaptorname/webadaptor. Si un alias DNS doit être utilisé avec le portail, ArcGIS Web Adaptor doit être configuré via l'alias DNS, à l'aide d'une URL telle que https://<dnsalias.domain.com>/<webadaptorname>/webadaptor.

  • <URL> : URL de la machine qui héberge Portal for ArcGIS. Incluez le nom de domaine complet de la machine dans l'URL, par exemple https://portal.domain.com:7443.

  • <Nomutilisateur> : nom d'utilisateur d'un compte qui dispose des privilèges d'administrateur sur Portal for ArcGIS. Vous utilisez généralement le compte d'administrateur initial que vous avez défini lors de la première configuration de votre portail.

  • <MotdePasse> : mot de passe d'un compte qui dispose des privilèges d'administrateur sur Portal for ArcGIS. Vous utilisez généralement le compte d'administrateur initial que vous avez défini lors de la première configuration de votre portail.

Exemple : ConfigureWebAdaptor.exe /m portal /w https://webadaptorhost.domain.com/webadaptorname/webadaptor /g https://portal.domain.com:7443 /u initialadmin /p secret

Après l'exécution de la commande, ArcGIS Web Adaptor est configuré de manière à être utilisé avec la machine qui héberge Portal for ArcGIS. Vous accédez désormais à votre portail par l'intermédiaire de l'URL d'ArcGIS Web Adaptor, au lieu du port 7443. L'URL se présente comme suit : https://webadaptorhost.domain.com/webadaptorname/home.

Si vous comptez configurer Portal for ArcGIS avec le serveur proxy de votre organisation, vous devez fournir certaines informations sur votre portail concernant le serveur proxy. Pour des instructions complètes, reportez-vous à la rubrique Configuration d'un serveur proxy avec votre portail.
